# 公司程式編碼原則（程式代號）

> **共用規範**：跨專案／跨模組之程式編碼原則請見 [程式編碼原則｜共用規範](程式編碼原則_共用規範.md)。本文件為**報關管理專案**之對應與範例，依該共用規範訂定。

## 1. 目的

建立統一的程式代號規則，供各專案共用，提升跨專案查找與溝通效率。**主程式代號（如 CUS110）** 用於**使用者授權之權限表**，對應選單／功能入口；作業內之報表、查詢、建檔、開窗等附屬功能不在此編列獨立代號。

## 2. 代號結構

- **模組代號**：三碼不重複英文縮寫（例：`CUS`）
- **功能代號**：三位數（例：`110`）
- **完整代號**：`<模組代號><功能代號>`（例：`CUS110`）

## 3. 百位數規劃（報關管理模組）

> 模組名稱：**報關管理**。整體作業分類：出口、進口、轉運、機場、SP、船用品。完整代號見 [報關管理模組與程式代號架構](../02_Analysis/2026-02-09_報關管理模組與程式代號架構.md)。

- 主作業與百位數對應**已定案**：
  - `1xx`：出口作業
  - `2xx`：進口作業
  - `3xx`：轉運作業
  - `4xx`：機場作業
  - `5xx`：SP 作業
  - `6xx`：船用品作業

## 4. 十位與個位規劃（權限表主程式）

每一作業僅編列四支主程式，供使用者授權權限表使用：

- `x00`：作業目錄／入口
- `x10`：案件建檔（其內成單、文件審核、帳務、運輸、聯絡單、結案等為子作業，以子編碼編列）
- `x20`：案件查詢（含增修）
- `x30`：案件歷史歸檔

匯出、統計等為程式延伸，以子編碼編列（如 `CUS120_Export01`），不納入權限表。

## 5. 使用原則

- 同一專案必須使用同一模組代號
- 同一功能的碼位意義需跨模組一致
- 新增功能以尾碼遞增（例：`CUS231`）
- 文件命名與程式代號保持一致
- 雛型 HTML 檔名建議：`YYYY-MM-DD_prototype_<程式代號>.html`（例：`2026-02-05_prototype_CUS110.html`）。CUS110＝出口案件建檔；CUS100 保留給出口作業目錄。
- 下拉清單建議使用「開窗輸入、自動收集」的標籤化方式
- 作廢規則如涉及對外編號，建議保留舊號避免空號，並重取系統編號

## 6. 範例（報關管理系統）

每作業四支主程式（權限表用）：

- **出口**：CUS100 目錄，CUS110 建檔，CUS120 查詢（含增修），CUS130 歷史歸檔
- **進口**：CUS200 目錄，CUS210 建檔，CUS220 查詢（含增修），CUS230 歷史歸檔
- **轉運**：CUS300 目錄，CUS310 建檔，CUS320 查詢（含增修），CUS330 歷史歸檔
- **機場**：CUS400 目錄，CUS410 建檔，CUS420 查詢（含增修），CUS430 歷史歸檔
- **SP**：CUS500 目錄，CUS510 建檔，CUS520 查詢（含增修），CUS530 歷史歸檔
- **船用品**：CUS600 目錄，CUS610 建檔，CUS620 查詢（含增修），CUS630 歷史歸檔

成單、文件審核、帳務、運輸、聯絡單、結案等為**建檔內子作業**，以子編碼編列（如 CUS110_xxx）。匯出、統計為程式延伸，子編碼。完整對照表見 [報關管理模組與程式代號架構](../02_Analysis/2026-02-09_報關管理模組與程式代號架構.md)。

## 7. 建檔內子作業與附屬功能編碼（不納入權限表）

- **建檔內子作業**（成單、文件審核、帳務、運輸、聯絡單輸出、結案）不編主程式代號，以**子編碼**編列（例：`CUS110_Order`、`CUS110_Report01`）。
- **報表、開窗、匯出、統計** 採 **主程式代號＋底線＋識別**，例：`CUS110_Report01`、`CUS120_Export01`；由**程式設計師**依團隊規則編列，不列入權限表清單。
