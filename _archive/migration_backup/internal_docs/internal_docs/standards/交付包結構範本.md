# 中央標準｜交付包結構範本

本文件定義**交付開發包**之目錄結構與各檔案用途，供 AI 與人工產出交付文件時依循。

---

## 0. 目錄命名原則

每支主程式（含依附開窗）為**獨立交付開發包**，目錄命名：

```
交付開發包_程式編碼程式名稱
```

**範例**：`交付開發包_CUS110出口作業`、`交付開發包_CUSSYS010表單組態維護`

每包內含**雛型最新版 HTML**，置於 `02_雛型/prototype/`。

---

## 1. 有 UI 之專案

```
交付開發包/
├── README.md                      # 必備：交付說明、版次、對應 Tag
├── .cursorrules                   # 必備：專案專用 AI 指令（公司標準 + 本專案客製）
├── 01_規格/
│   ├── PRD_本階段範圍摘要.md      # 必備：本階段功能摘要（非全文）
│   ├── 程式代號與權限矩陣.md      # 必備：模組與主程式、權限矩陣
│   ├── 欄位名稱對照表.md         # 必備：本階段涉及之欄位（中英、縮寫、用途）
│   ├── 情境與驗收說明.md         # 必備：情境說明 + 驗收條件清單
│   ├── 開發要求與資安清單.md      # 必備：資安與技術要求精華
│   └── 子程式清單.md             # 必備：主程式之子程式（子編碼）清單；依 [子程式清單與實作建議｜治理規範](子程式清單與實作建議_治理規範.md) 產出
├── 02_雛型/
│   ├── prototype/                 # 必備：本階段雛型 HTML（定稿版）
│   └── 操作步驟說明.md           # 必備：雛型操作步驟
└── 03_附錄/
    ├── 工程師開工指引.md           # 必備：建議進行步驟、AI Prompt、建議環境、準備事項；依 [工程師開工指引｜治理規範](工程師開工指引_治理規範.md)
    ├── schema_參考.md              # 建議必備（有 UI）：本程式範圍之 TABLE 結構；依 [TABLE與API設計_AI產出指引](TABLE與API設計_AI產出指引.md)，利於開工即對齊
    ├── API_規範.md                 # 建議必備（有 UI）：本程式範圍之 API 路徑與請求／回應；同上，利於 AI 與工程師實作
    ├── 依賴介面摘要.md             # 選用：有依賴他程式／API 時；路徑、必填參數、回傳要點或連結至他程式 API_規範
    └── 流程圖與架構.md             # 選用
```

### 1.1 各檔案說明

| 檔案 | 用途 | 範本 |
| :--- | :--- | :--- |
| README.md | 交付說明、版次、對應 Tag、**依賴清單**、**技術棧／認證**、本階段範圍、取得更新方式 | Tpl_交付開發包_README |
| .cursorrules | AI 讀取時之行為與引用；含公司標準精華 + 本專案程式代號、欄位、雛型路徑 | cursorrules_範本 |
| PRD_本階段範圍摘要.md | 本階段功能列表、關鍵流程；不納入 Discovery、歷史分析 | Tpl_PRD_本階段範圍摘要 |
| 程式代號與權限矩陣.md | 本階段涉及之模組與主程式、權限矩陣表；命名依 [程式編碼原則](../../程式編碼原則_共用規範.md) | Tpl_程式代號與權限矩陣_交付用 |
| 欄位名稱對照表.md | 本階段表單與報表欄位：中文、英文、縮寫、用途 | Tpl_欄位名稱對照表_交付用 |
| 情境與驗收說明.md | 使用情境、驗收條件清單（可檢核格式） | Tpl_情境與驗收說明_交付用 |
| 開發要求與資安清單.md | 資安、技術、合規要求精華 | Tpl_開發要求與資安清單 |
| 子程式清單.md | 主程式之子程式（子編碼）清單；雛型對應、實作建議；依治理規範產出 | Tpl_子程式清單_交付用 |
| prototype/*.html | 定稿雛型，檔名依 `YYYY-MM-DD_prototype_<程式代號>.html` | — |
| 操作步驟說明.md | 雛型操作步驟、區塊說明 | — |
| 03_附錄/工程師開工指引.md | 建議進行步驟、AI Prompt、建議環境、準備事項；供工程師開工 | Tpl_工程師開工指引 |

## 2. 無 UI 之批次程式

```
交付開發包/
├── README.md
├── .cursorrules
├── 01_規格/
│   ├── PRD_本階段範圍摘要.md
│   ├── 程式代號與權限矩陣.md
│   ├── 欄位名稱對照表.md
│   ├── 批次規格/                  # 取代 02_雛型
│   │   ├── BATCH001_每日匯出.md
│   │   ├── BATCH002_月結轉檔.md
│   │   └── ...
│   ├── 情境與驗收說明.md
│   └── 開發要求與資安清單.md
└── 02_附錄/
    └── 流程圖與架構.md
```

### 2.1 批次規格說明

- 每支批次一檔，檔名：`<程式代號>_<簡述>.md`
- 內容格式見 [批次規格範本](批次規格範本.md)
- 涵蓋：觸發條件、輸入、處理邏輯、輸出、錯誤處理、驗收條件

## 3. 產出原則

| 原則 | 說明 |
| :--- | :--- |
| **精簡** | 僅納入本階段要開發之範圍，不納入 Discovery、歷史分析 |
| **可讀** | 工程團隊 AI 僅讀此包即可開發，不需讀取完整需求 Repo |
| **子程式清單** | 有 UI 之主程式交付包須含 `01_規格/子程式清單.md`；依 [子程式清單與實作建議｜治理規範](子程式清單與實作建議_治理規範.md) 產出與檢查 |
| **可追溯** | README 註明對應需求 Repo 之 Tag／commit、版次 |
| **可檢核** | 驗收條件為結構化格式，供 AI 或人工檢核 |
| **依賴明列** | README 或 01_規格 列出依賴；有依賴時 03_附錄 建議含依賴介面摘要或連結 |
| **技術棧引用** | README 或 03_附錄 註明技術棧與認證（依專案決策備忘或簡要列出） |
