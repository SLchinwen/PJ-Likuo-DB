# 中央標準｜子程式清單與實作建議｜治理規範

本規範定義**產出交付開發包時**對子程式清單之**檢查與產出要求**，以及**AI 實作時**如何依子程式清單展開工作包。

---

## 1. 適用時機

| 情境 | 適用 |
| :--- | :--- |
| **產出交付開發包** | 有 UI 之主程式交付包，須含 `01_規格/子程式清單.md` |
| **AI 實作展開** | AI 依交付包內子程式清單與實作建議，拆解任務、估工、實作 |

---

## 2. 產出交付包時之檢查與產出

### 2.1 必備檢查

產出或組裝交付開發包時，**必須**完成下列檢查：

| 檢查項 | 說明 | 若未通過 |
| :--- | :--- | :--- |
| **子程式清單存在** | `01_規格/子程式清單.md` 已產出並置於交付包內 | **提醒**：請依雛型與需求產出子程式清單；或由 AI **自動產出** |
| **雛型對應完整** | 子程式清單內各子程式有對應雛型區塊／按鈕／開窗 | 補齊雛型對應欄位 |
| **編碼格式正確** | 子程式代號格式為 `<主程式>_<識別>`，識別小寫、底線分隔 | 修正編碼格式 |

### 2.2 自動產出流程（AI 執行）

當需求方要求「產出交付開發包」且**子程式清單尚未存在**時，AI 應：

1. **依雛型檢視**：讀取 `02_雛型/prototype/*.html` 與 `02_雛型/操作步驟說明.md`
2. **依需求對照**：讀取 PRD 摘要、情境與驗收說明
3. **產出子程式清單**：依 [Tpl_子程式清單_交付用](../../00_Templates/Tpl_子程式清單_交付用.md) 產出 `01_規格/子程式清單.md`
4. **納入交付包**：將產出之 `子程式清單.md` 置於 `01_規格/` 內
5. **更新 README、使用指引、.cursorrules**：於目錄結構與必讀文件中加入子程式清單

### 2.3 提醒語句（人工產出時）

若由**人工**產出交付包，AI 可於完成其他檔案產出後，輸出下列提醒：

> **治理檢查提醒**：依 [子程式清單與實作建議｜治理規範](standards/子程式清單與實作建議_治理規範.md)，交付包須含 `01_規格/子程式清單.md`。若尚未產出，請依雛型與需求檢視產出，或要求 AI 代為產出。

---

## 3. 子程式清單內容要求

子程式清單須含下列章節（詳見 [Tpl_子程式清單_交付用](../../00_Templates/Tpl_子程式清單_交付用.md)）：

| 章節 | 說明 |
| :--- | :--- |
| 編碼原則 | 格式、權責、權限表說明 |
| 主作業子程式 | 建檔內子作業、查詢、表列等 |
| 開窗／附屬功能 | 彈窗、查詢開窗、附屬表單 |
| 報表／輸出 | 列印、匯出、批次輸出 |
| 雛型開窗一覽 | 雛型元素 ID 與子程式對照 |
| **實作建議**（選用） | 實作順序、依賴關係、AI 展開指引；見 [子程式實作建議｜標準](子程式實作建議_標準.md) |

---

## 4. AI 實作時之展開

AI 於**實作階段**讀取交付包時，應：

1. **讀取子程式清單**：`01_規格/子程式清單.md`
2. **讀取實作建議**：交付包內若有「實作建議」章節，或引用 [子程式實作建議｜標準](子程式實作建議_標準.md)
3. **依子程式拆解任務**：每個子程式可對應一組實作任務（元件、API、服務）
4. **依建議順序實作**：若有依賴關係，依建議順序展開；無則可平行

詳見 [子程式實作建議｜標準](子程式實作建議_標準.md)。
