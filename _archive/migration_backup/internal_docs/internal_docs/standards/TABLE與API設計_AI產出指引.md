# 中央標準｜TABLE 與 API 設計｜AI 產出指引

本文件定義**非整合既有系統**情境下，AI 如何依目前規畫產出 **TABLE schema** 與 **API 規範**，以及**開發順序**建議。若專案需整合既有系統，則以既有 schema 與 API 為準。

---

## 1. TABLE 設計（schema）

### 1.1 產出權責

| 情境 | 產出方式 |
| :--- | :--- |
| **新系統、無既有 DB** | AI 依**欄位對照表、PRD、子程式清單**產出最佳 TABLE 結構與 schema 參考 |
| **整合既有系統** | 以既有 schema 為準；AI 可產出**對照與差異分析** |

### 1.2 AI 產出 schema 之依據

- **欄位名稱對照表**：TABLE、欄位命名依此對齊
- **程式編碼原則**：TABLE 命名依 [程式編碼原則｜共用規範](../程式編碼原則_共用規範.md) §4～§5
- **PRD 與子程式清單**：辨識實體、關聯、主鍵、外鍵

### 1.3 產出格式

AI 可產出下列格式之一：

- **DDL**（SQL CREATE TABLE）
- **Markdown 表格式**（TABLE 名、欄位、型別、主鍵、外鍵、索引）
- **ER 圖描述**（Mermaid 或文字說明）

放置於交付包 `03_附錄/schema_參考.md` 或專案約定目錄。

### 1.4 環境設定建議

若需配合設定**資料庫環境**時，可建議：

| 項目 | 建議 |
| :--- | :--- |
| **資料庫** | PostgreSQL、SQL Server、MySQL 等任選；依專案既有規範 |
| **連線** | 使用環境變數（如 `DATABASE_URL`）管理連線字串 |
| **Migration** | 建議使用 schema 版本管理（如 Flyway、Entity Framework Migrations） |
| **種子資料** | 若需初始資料，可產出 seed SQL 或 JSON |

## 2. API 規範

### 2.1 產出權責

| 情境 | 產出方式 |
| :--- | :--- |
| **新系統** | AI 依**子程式清單、欄位對照表、情境**產出 API 規範 |
| **既有 API** | 以既有規範為準 |

### 2.2 AI 產出 API 規範之依據

- **子程式清單**：每個子程式對應一組 API（查詢、新增、更新、動作）
- **子程式實作建議｜標準**：API 路徑 pattern、命名慣例
- **欄位對照表**：請求／回應欄位命名

### 2.3 產出內容

- **Endpoint 路徑**：`GET /api/{主程式}/{資源}/list` 等
- **請求／回應格式**：JSON 結構、欄位型別
- **錯誤碼**：建議統一格式（如 `{ code, message }`）
- **權限**：依權限矩陣表註記

放置於交付包 `03_附錄/API_規範.md` 或專案約定目錄。

## 3. 開發順序建議

### 3.1 原則

- **優先平行**：無依賴時可平行開發
- **依賴分組**：依賴性強時分為一組，組內依序、組間可平行

### 3.2 本專案建議順序（合群報關管理系統）

| 組別 | 程式 | 依賴 | 說明 |
| :--- | :--- | :--- | :--- |
| **A 組（先開發）** | CUS010 | — | 客戶主檔與聯絡人；基礎主檔，其他程式會引用 |
| **A 組** | CUSSYS010 | — | 表單組態維護；form_config 供 CUS110 等建檔使用 |
| **B 組（A 組完成後）** | CUS110 | CUS010、form_config | 出口建檔；需客戶開窗、form_config 選項 |
| **C 組（B 組完成後）** | CUS710 | CUS110（及未來進口等） | 運務查詢；資料來自 CUS110 等建檔之運務需求 |

### 3.3 平行與依賴關係圖

```
A 組（可平行）：
  CUS010 ────────┐
                 ├──→ B 組：CUS110
  CUSSYS010 ─────┘

B 組完成後：
  CUS110（及未來進口等）──→ C 組：CUS710
```

### 3.4 調整建議

- 若 CUS110 可先以**固定選項**或 mock form_config 開發，則 CUSSYS010 與 CUS110 可部分平行
- CUS710 可於 CUS110 有運務資料後再開發，或先以 mock 資料開發 UI
