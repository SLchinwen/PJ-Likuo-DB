# 需求到交付包｜整併與版本說明

本文件為**從最前端需求、公司背景到釋出協助開發包**整併作業之摘要：檢視範圍、遺漏補齊、去重與衝突處理、新版本一覽。供後續維護與新專案引用時對照。

---

## 1. 檢視範圍

- **流程**：公司背景 → Discovery → RFP → 主程式架構 → PRD（四項必備）→ 程式編碼原則 → 雛型／批次規格 → 交付門檻 → 產出交付開發包 → 釋出給工程師＋AI 協作開發 → 驗收與迭代。
- **產出**：生命週期總覽、缺口補齊、程式編碼原則統一、治理總則與索引更新、_Inbox 與 internal_docs 單一真相來源標註。

## 3. 去重與衝突處理

| 項目 | 處理方式 |
| :--- | :--- |
| **程式編碼原則（三份）** | **程式編碼原則_共用規範** 與 **模組與程式代碼編碼原則**：訂為**併用**；文首互相引用。**模組代號**統一為**建議 2～3 碼英文大寫**（原共用規範為三～五碼，已對齊模組與程式代碼）。**coding_standard_program_codes** 維持為報關專案實例，引用共用規範。 |
| **internal_docs 與 _Inbox 重複** | 明訂 **internal_docs** 為單一真相來源；_Inbox/README 新增「歸檔後以 internal_docs 為準，請勿雙版本並存」；GOV 總則 §2.3、§7 標註 internal_docs 與生命週期總覽。 |
| **GOV 與公司標準規範關係** | GOV 總則 §2.3 新增「流程與內部標準」、§7 補 internal_docs 與生命週期總覽；GOV 為母體目錄與 YAML／命名，公司標準規範為需求與開發協作細則，兩者互補。 |
