## 2. 各專案如何引用

### 2.1 產品開發

- 中央標準權重較高
- 交付包可做成「產品版型」，各版本僅改差異部分
- 引用：複製 `standards/` 範本至產品 Repo，或於交付包 README 註明引用版次

### 2.2 客製專案

- 專案客製權重較高
- 每個專案產出專屬交付包，引用中央標準為底層
- 引用：交付時將中央標準精華 + 專案客製合併寫入交付包 `.cursorrules`

### 2.3 新專案啟動

1. 複製 [協同開發文件治理流程](../協同開發_文件治理流程.md) 至新專案 `internal_docs/`
2. 複製 `standards/` 下所需範本，或建立連結引用
3. 依 [交付包結構範本](交付包結構範本.md) 建立交付開發包目錄
4. 依 `00_Templates/` 下交付用範本產出各檔案
