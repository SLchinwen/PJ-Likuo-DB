# 中央標準｜子程式實作建議

本文件供 **AI 實作時**參考，依子程式清單**展開工作包**、拆解任務、建議實作順序與命名。各交付包之子程式清單可引用本標準，或於清單內附加專案專屬之實作建議。

---

## 1. 子程式類型與實作對應

| 子程式類型 | 雛型對應 | 實作建議 | 常見產物 |
| :--- | :--- | :--- | :--- |
| **搜尋／查詢** | 搜尋列、查詢按鈕 | 表單元件 + 查詢 API + 結果狀態 | `SearchForm`、`useQuery`、`GET /api/...` |
| **表列／清單** | 表格、分頁 | 表格元件 + 列表 API + 排序／篩選 | `DataTable`、`GET /api/.../list` |
| **新增／編輯** | 表單、儲存按鈕 | 表單元件 + 驗證 + 新增／更新 API | `Form`、`POST`／`PUT` |
| **開窗（Modal）** | 彈出視窗 | Modal 元件 + 表單／查詢 + 選取回傳 | `Modal`、`onSelect` callback |
| **按鈕動作** | 單一按鈕（成單、結案、作廢） | 按鈕 + 確認 + 單次 API | `handleAction`、`POST` |
| **報表／匯出** | 列印、匯出按鈕 | 匯出 API + 檔案下載／列印 | `GET /api/.../export`、`Blob` |

---

## 2. 實作順序建議

### 2.1 依賴優先

| 順序 | 類型 | 說明 |
| :--- | :--- | :--- |
| 1 | **基礎查詢／表列** | 主畫面查詢與結果表列，為後續操作之入口 |
| 2 | **開窗（查詢選取）** | 客戶查詢、船期查詢等開窗，供主表單帶入 |
| 3 | **新增／編輯表單** | 主檔維護、子作業表單 |
| 4 | **按鈕動作** | 成單、結案、作廢等狀態變更 |
| 5 | **報表／匯出** | 列印、匯出 Excel，依查詢結果產出 |

### 2.2 平行可實作

- 多個**獨立開窗**可平行
- **報表／匯出**與主流程無強依賴時可平行
- 同一主程式內之**不同區塊**（如版次管理 vs 選項維護）可平行

## 3. 命名與結構建議

### 3.1 前端元件

| 子程式代號 | 建議元件命名 | 說明 |
| :--- | :--- | :--- |
| `XXX_search` | `XxxSearchForm`、`useXxxSearch` | 搜尋表單與 hook |
| `XXX_result_list` | `XxxResultTable`、`XxxList` | 結果表列 |
| `XXX_modal` | `XxxModal`、`XxxSelectModal` | 開窗元件 |
| `XXX_add` / `XXX_edit` | `XxxForm`、`XxxEditForm` | 新增／編輯表單 |
| `XXX_export` | `handleXxxExport`、`XxxExportButton` | 匯出邏輯或按鈕 |

### 3.2 API 路徑

| 子程式類型 | 建議路徑 pattern | 範例 |
| :--- | :--- | :--- |
| 查詢列表 | `GET /api/{主程式}/{資源}/list` | `GET /api/cus110/orders/list` |
| 單筆查詢 | `GET /api/{主程式}/{資源}/{id}` | `GET /api/cus110/orders/123` |
| 新增 | `POST /api/{主程式}/{資源}` | `POST /api/cus110/orders` |
| 更新 | `PUT /api/{主程式}/{資源}/{id}` | `PUT /api/cus110/orders/123` |
| 動作 | `POST /api/{主程式}/{資源}/{id}/{action}` | `POST /api/cus110/orders/123/close` |
| 匯出 | `GET /api/{主程式}/{資源}/export` | `GET /api/cus710/transports/export` |

### 3.3 檔案結構（建議）

```
features/{主程式}/
├── components/          # 主程式共用元件
│   ├── XxxSearchForm.tsx
│   ├── XxxResultTable.tsx
│   └── modals/
│       ├── ClientSearchModal.tsx   # 對應 CUS110_client_search
│       └── ShipScheduleModal.tsx   # 對應 CUS110_ship_schedule
├── hooks/
│   └── useXxxQuery.ts
└── api/
    └── xxxApi.ts
```

## 4. 任務拆解範例

以 `CUS110_client_search`（客戶查詢開窗）為例：

| 任務 | 說明 | 產物 |
| :--- | :--- | :--- |
| 1 | 建立 Modal 骨架 | `ClientSearchModal.tsx` |
| 2 | 搜尋表單（客戶名稱） | 表單欄位 + 查詢按鈕 |
| 3 | 呼叫 CUS010 或共用 API | `GET /api/cus010/parties/search?name=xxx` |
| 4 | 結果表列與選取 | 點選列 → 帶入主表單統編、聯絡人 |
| 5 | 關閉 Modal 回傳選取值 | `onSelect(party)` |

## 5. 驗收對應

每個子程式應對應至少一項驗收條件（見 `情境與驗收說明.md`）。實作時：

- 依子程式代號找到對應驗收條件
- 完成實作後，逐條自檢
- 若有「雛型對應」欄位，確保畫面與雛型一致
