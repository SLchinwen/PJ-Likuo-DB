# 模組與程式代碼編碼原則（程式文件生成參考）

本文件為**內部文件**，供**程式文件生成、PRD 撰寫、雛型命名、框架／TABLE／API 設計**時依循。與 [程式編碼原則｜共用規範](程式編碼原則_共用規範.md) **併用**：共用規範為權限表與主程式／子編碼原則之總覽；**本文件為格式約束、禁止事項、雛型／TABLE／API 對齊與程式文件生成檢查之依據**。各專案須先訂定模組代號與主作業百位數後，再依本原則編列主程式與子編碼。

---

## 1. 用語定義（嚴謹）

| 用語 | 定義 |
| :--- | :--- |
| **模組代號** | 二至三碼英文縮寫，代表單一業務模組或子系統，同一專案內不重複。 |
| **功能代號** | 三位阿拉伯數字，百位數對應主作業類別，十位個位對應功能類型（目錄／建檔／查詢／歸檔）。 |
| **主程式代號** | 模組代號＋功能代號，對應選單／功能入口，**須登錄於使用者授權權限表**。 |
| **子編碼** | 主程式代號後接底線與識別，對應建檔內子作業、報表、開窗、匯出等，**不登錄權限表**。 |
| **主作業** | 由專案定義之業務分類（如出口、進口、訂單、出貨），對應一組百位數（1xx、2xx…）。 |

---

## 2. 代號結構與格式約束

### 2.1 主程式代號格式

| 組成 | 格式約束 | 範例 | 備註 |
| :--- | :--- | :--- | :--- |
| **模組代號** | 2～3 碼，**僅英文大寫**（A–Z），同一專案內唯一 | `CUS`、`ORD`、`KID` | 由需求方／專案訂定，不得與他模組重複 |
| **功能代號** | **固定 3 位數**，僅數字 0–9；百位＝主作業，十位個位＝功能類型 | `100`、`110`、`230` | 見 §3 十位個位定義 |
| **完整主程式代號** | `<模組代號><功能代號>`，**無底線、無空格、無特殊字元** | `CUS110`、`ORD210` | 用於權限表、選單、雛型檔名、程式與文件命名 |

**禁止**：模組代號使用小寫、數字或符號；功能代號非三位數或含非數字字元；主程式代號中間插入底線或連字號。

### 2.2 子編碼格式

| 組成 | 格式約束 | 範例 | 備註 |
| :--- | :--- | :--- | :--- |
| **主程式代號** | 須為已存在之主程式代號 | `CUS110` | 不可憑空造碼 |
| **底線** | 單一底線 `_` 連接 | — | 僅允許一個底線 |
| **識別** | 英文、數字或合理縮寫；建議 **PascalCase 或序號**，避免空格與特殊字元 | `Order`、`Report01`、`Export01` | 由程式設計師依團隊規則編列，專案內建議留存清單 |

**完整子編碼**：`<主程式代號>_<識別>`，例：`CUS110_Order`、`CUS120_Export01`。  
**禁止**：子編碼登錄於使用者權限表；識別使用空格或 `-` 等多字元連接符（僅允許單一 `_`）。

---

## 3. 權限表主程式原則（每作業四支）

**僅下列四類**得編列為主程式代號，登錄於使用者授權之權限表；對應選單／功能入口，供權限控管使用。其餘功能一律以子編碼延伸，**不得**另編主程式代號。

| 十位個位 | 功能類型 | 說明 | 權限表登錄 |
| :--- | :--- | :--- | :--- |
| **x00** | 作業目錄 | 該主作業之入口／選單頁 | ✅ 登錄 |
| **x10** | 案件建檔 | 案件新增與維護；其內成單、審核、帳務、運輸、聯絡單、結案等為**子作業**，以子編碼編列 | ✅ 登錄 |
| **x20** | 案件查詢（含增修） | 案件查詢與增刪修；匯出、統計以子編碼延伸 | ✅ 登錄 |
| **x30** | 案件歷史歸檔 | 歷史歸檔與歷史資料查詢 | ✅ 登錄 |

- 每一**主作業**對應一組**百位數**（1xx、2xx、3xx…），並編列上述四支主程式（x00、x10、x20、x30）。
- **同一專案內**，十位個位之意義必須一致：所有 x00 皆為目錄、x10 皆為建檔、x20 皆為查詢含增修、x30 皆為歷史歸檔。
- 匯出、統計、報表、開窗、查詢子畫面等**不得**另編主程式代號，僅能以子編碼延伸。

---

## 4. 子編碼原則（不納入權限表）

下列項目**不**給予獨立主程式代號，由**程式設計師**依本規範與團隊規則以**子編碼**編列，**不列入**使用者授權權限表。

### 4.1 建檔內子作業

- 建檔程式（x10）內之流程步驟：成單與編號、文件審核、帳務、運輸安排、聯絡單輸出、結案等。
- 編碼方式：`<主程式代號>_<識別>`，例：`CUS110_Order`、`CUS110_DocAudit`、`CUS110_Report01`。

### 4.2 報表、開窗、查詢子畫面

- 作業內之報表、開窗、查詢子畫面等附屬功能。
- 編碼方式：`<主程式代號>_<識別>`，例：`CUS110_Report01`、`CUS120_Query01`。

### 4.3 匯出、統計（程式延伸）

- 查詢或列表之匯出、統計等延伸功能。
- 編碼方式：`<主程式代號>_<識別>`，例：`CUS120_Export01`、`CUS120_Report01`。

### 4.4 編列權責

- **主程式代號**（權限表用）：由**需求方／專案**訂定，寫入各專案之「模組與程式代號架構」或 PRD，**不得**由開發方自行新增主程式代號。
- **子編碼**：由**程式設計師**依本規範與團隊規則編列，不列入權限表；建議於專案內留存**子編碼清單**或命名規則說明，供程式文件生成時對照。

---

## 5. 百位數規劃（各專案自訂）

- **百位數**由各專案／模組自訂，對應該專案之**主作業分類**（例如：1xx 出口、2xx 進口、3xx 訂單…）。
- **新增主作業**時，以**未使用之百位數**遞增編列，不得跳號或重複使用已作廢之百位數（除非專案明訂回收規則）。
- 同一專案內，主作業與百位數之對應須**單一且穩定**，並記載於模組／程式代號架構文件。

## 6. 雛型、檔案與程式命名（程式文件生成時必循）

### 6.1 雛型 HTML 檔名

| 規則 | 格式 | 範例 |
| :--- | :--- | :--- |
| **主程式雛型** | `YYYY-MM-DD_prototype_<主程式代號>.html` | `2026-02-05_prototype_CUS100.html` |
| **日期** | 西元年月日，以連字號分隔 | 雛型定稿或發布日 |
| **程式代號** | 僅主程式代號，無子編碼 | 子功能雛型可併入主程式或另約定 |

**禁止**：檔名使用小寫程式代號、省略日期、或與權限表主程式代號不一致。

**對外發布至 MkDocs**：生成對外文件時，雛型**最新版**更新至 MkDocs；對外雛型檔名**一律用「程式代號+程式名稱」**、**不分版次**（格式 `<程式代號>_<程式名稱英文或縮寫>.html`）；版次於 HTML 內容或標題呈現。詳見 [雛型產出規範與指引](雛型產出規範與指引.md) §7.2。

### 6.2 文件與程式命名對齊

- **文件**（PRD、設計書、操作手冊）中出現之程式名稱、畫面代號、功能代號，須與本原則之**主程式代號／子編碼**一致。
- **程式**（前端／後端模組、API、畫面 ID）命名須與程式代號對齊，以便追溯與權限控管；若技術慣例不同（如 kebab-case），須在專案內建立**對照表**並註明於程式文件。

### 6.3 框架、TABLE、API 命名對齊（PRD 與程式文件生成時）

依公司 e化 標準，PRD 產出之**模組與主程式代碼**為**框架、資料表（TABLE）、API** 命名與權限規劃之依據。程式文件生成時應遵守：

- **框架／模組路徑**：可依主程式代號或模組代號分層，例：`CUS/110`、`CUS110`，與權限表主程式一致。
- **資料表（TABLE）**：建議以業務意義命名，並在設計文件註明對應之**主程式代號**或模組，例：`CUS_MAIN`、`CUS_DETAIL`，或專案約定之前綴＋業務名稱。
- **API**：建議路徑或代碼含模組／程式識別，例：`/api/cus/110/...` 或 `CUS110_Query`，與主程式代號對齊，便於權限與稽核對應。
- 上述對齊方式若有專案例外，須於**內部設計文件**中載明並與本原則對照。

## 7. 作廢與刪除規則

- **對外有編號**之案件或單據：建議採**作廢保留編號**、系統單號重取，**避免空號**，以利查詢與稽核。
- **主程式代號**：一經登錄權限表與文件，**不得**隨意刪除或改號；若功能整併或廢止，應訂定「作廢保留」或「遷移對照」並更新權限表與文件。
- **子編碼**：由程式設計師編列，廢止時可於專案內註記，不影響權限表。

---

## 8. 禁止事項（嚴謹遵守）

| 禁止項目 | 說明 |
| :--- | :--- |
| 主程式代號跳號或重複 | 同一模組內主程式代號不得重複；百位數內 x00/x10/x20/x30 須齊備，不得僅編部分。 |
| 子編碼登錄權限表 | 僅主程式代號（x00、x10、x20、x30）登錄；子編碼不得作為權限控管單位。 |
| 模組代號混用小寫或數字 | 模組代號僅英文大寫，2～3 碼。 |
| 功能代號非三位數 | 功能代號固定 3 位數，不足補零（如 01 應為 010，僅在十位個位為 00/10/20/30 時使用三位數）。 |
| 雛型或文件與代號不一致 | 雛型檔名、文件內程式清單、API/ TABLE 設計須與本原則對齊，不得自創代號。 |
| 未訂主程式即產出 PRD | 須**先**建立模組與主程式代號清單，再撰寫功能作業 PRD；見內部「標準分析產出要求」。 |

---

## 9. 程式文件生成時之檢查清單

產出或審核**程式相關文件**（PRD、雛型清單、權限矩陣、程式清單、API/ TABLE 設計）時，請確認：

- [ ] **模組代號**：2～3 碼英文大寫，專案內唯一，與架構文件一致。
- [ ] **主程式代號**：格式為 `<模組代號><三位數>`，每主作業僅四支（x00、x10、x20、x30），且已登錄或將登錄權限表。
- [ ] **子編碼**：僅使用 `<主程式代號>_<識別>`，未登錄權限表；識別格式符合團隊約定。
- [ ] **雛型檔名**：主程式雛型為 `YYYY-MM-DD_prototype_<主程式代號>.html`，與權限表一致。
- [ ] **文件與程式命名**：文件內程式清單、畫面代號、API/ TABLE 命名與本原則對齊，並可追溯至主程式代號或子編碼。
- [ ] **權限表**：僅含主程式代號，不含子編碼；權限矩陣之程式代號與本原則一致。
- [ ] **作廢與變更**：若有代號作廢或整併，已註記並更新權限表與相關文件。

## 10. 專案套用方式（摘要）

1. **選定模組代號**：2～3 碼英文大寫，專案內唯一。
2. **訂定主作業與百位數**：列出所有主作業，分配百位數（1xx、2xx…）。
3. **產出主程式代號清單**：每一主作業編列四支主程式（x00、x10、x20、x30），寫入架構或 PRD。
4. **權限表**：僅登錄上述主程式代號。
5. **子編碼**：由程式設計師依 §4 編列，並於專案內留存清單或命名規則。
6. **程式文件生成**：雛型、API、TABLE、文件內程式清單皆依本原則對齊並通過 §9 檢查清單。

## 11. 範例摘要（報關管理模組）

| 項目 | 內容 |
| :--- | :--- |
| 模組代號 | `CUS` |
| 主作業與百位數 | 1xx 出口、2xx 進口、3xx 轉運、4xx 機場、5xx SP、6xx 船用品 |
| 主程式範例（出口） | CUS100 目錄、CUS110 建檔、CUS120 查詢（含增修）、CUS130 歷史歸檔 |
| 主程式範例（進口） | CUS200、CUS210、CUS220、CUS230；其餘類推 |
| 子編碼範例 | 建檔內：`CUS110_Order`、`CUS110_DocAudit`；報表：`CUS110_Report01`；匯出：`CUS120_Export01` |
| 雛型檔名範例 | `2026-02-05_prototype_CUS100.html` |

不同專案或模組：依同原則訂定**模組代號**與**百位數對應**後，套用 §3 之主程式結構即可。
