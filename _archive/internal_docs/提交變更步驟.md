# 提交變更步驟（可順利提交的標準程序）

目前 **main 未設分支保護**，可直接在 main 上提交並推送。若日後啟用 main 保護，則須改為「開分支 → PR → 合併」（見下方「二、若 main 已設保護」）。

---

## 一、直接推送到 main（目前適用）

在專案資料夾開啟**終端機**，依序執行：

### 步驟 1：確認在專案目錄並取得最新 main

```powershell
cd "C:\github\SL_Client_Case\HER _CHYUN"
git fetch origin
git checkout main
git pull origin main
```

### 步驟 2：暫存要提交的檔案

```powershell
git add docs/index.md
```

（多個檔案：`git add docs/index.md internal_docs/協作與治理規範.md` 或 `git add .`）

### 步驟 3：提交（寫入 commit 訊息）

```powershell
git commit -m "docs: 更新專案總攬（請依當次變更修改）"
```

### 步驟 4：推送到 main

```powershell
git push origin main
```

推送成功後，GitHub Actions 會自動建置並部署（若已設定），對外文件更新即上線。

---

## 二、若 main 已設保護（須透過 PR 時）

若 GitHub 已啟用 main 分支保護，則須改為開分支、推送分支、開 PR、合併：

1. 從 main 開新分支：`git checkout -b docs/update-xxx`
2. 修改、`git add`、`git commit`
3. 推送分支：`git push -u origin docs/update-xxx`
4. 到 GitHub 開 **Pull Request**（base: main，compare: 您的分支）
5. 審核通過後 **Merge**，再本機 `git checkout main`、`git pull origin main`

---

## 三、若出現 .git/index.lock 錯誤

在專案目錄執行（刪除鎖檔後再重試提交）：

```powershell
Remove-Item -Force ".git\index.lock" -ErrorAction SilentlyContinue
```

然後從**步驟 3**（git add）重新做一次。

---

## 四、用 Cursor 圖形介面時（備用）

目前可直接推 main 時：

1. 確認左下角分支為 **main**
2. **Source Control** → 對要提交的檔案點 **+**（暫存）
3. 在「訊息」框輸入 commit 訊息 → 點 **提交** 或 Ctrl+Enter
4. 點 **同步變更**（Push）→ 會推送到 main

若提交時出現 **index.lock**，關閉 Cursor 後刪除 `.git\index.lock`，再開 Cursor 重試；或改用「一、命令列」流程。
