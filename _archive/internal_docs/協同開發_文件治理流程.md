# 協同開發｜文件治理流程

本文件定義**需求方與工程團隊**協同開發時之**文件產出時機、交付包結構、迭代流程、驗收條件**，供 AI 與人工參考產出交付文件。適用於**有 UI 之 e化 專案**與**無 UI 之批次程式**。

---

## 1. 適用範圍

| 項目 | 說明 |
| :--- | :--- |
| **適用情境** | 需求方持有需求 Repo、工程團隊在自有專案開發；採用 AI 輔助產出與檢核 |
| **適用專案** | 產品開發、客製專案、有 UI 之表單作業、無 UI 之批次程式 |
| **目的** | 減少工程團隊 AI 需讀入之文件量；雛型驗證後再交付開發；結構化驗收供 AI 檢核 |

---

## 2. 整體流程總覽

```
【需求方階段】
訪談 → RFP、PRD（四項必備）、流程圖、程式清單
         ↓
與客戶討論確認
         ↓
雛型產出（有 UI）／批次規格產出（無 UI）
         ↓
客戶驗證 → 客戶回饋處理 → 定稿
         ↓
【交付門檻】PRD 確認 + 雛型／規格定稿
         ↓
產出「交付開發包」（精簡、本階段範圍）
         ↓
授權／交付給工程團隊（Tag、壓縮包或指定目錄）
         ↓
【工程團隊階段】
在自有專案開發，AI 僅讀交付開發包
         ↓
【驗收】
依驗收條件檢核（人工或 AI 輔助）
         ↓
【迭代】
客戶回饋／需求變更 → 需求變更單 → 更新 PRD／雛型
         ↓
產出「變更補充包」或「新階段交付包」→ 通知工程團隊
```

---

## 3. 各階段文件產出與放置

### 3.1 需求方內部階段（不交付工程團隊）

| 階段 | 產出文件 | 放置目錄 | 說明 |
| :--- | :--- | :--- | :--- |
| 訪談 | 訪談紀錄、客戶資料卡、需求展開 | `01_Discovery/` | 內部參考 |
| 分析 | RFP、流程圖、程式清單 | `02_Analysis/` | 與客戶討論用 |
| PRD | 模組與主程式代碼、權限矩陣、情境說明、欄位對照表 | `02_Analysis/` | 四項必備，見 [文件治理規範](docs_治理規範.md) §6 |
| 雛型 | 雛型 HTML、操作步驟說明 | `02_Analysis/`、`docs/prototype/` | 有 UI 專案 |
| 批次規格 | 輸入／輸出／處理邏輯規格 | `02_Analysis/` | 無 UI 批次程式 |
| 客戶驗證 | 客戶回饋、進版文件、需求變更 | `02_Analysis/` | 定稿前迭代 |

### 3.2 交付門檻

**交付開發包之產出時機**：以下兩項皆完成後

1. **PRD 確認**：四項必備齊備，與客戶對齊
2. **雛型定稿**（有 UI）或 **批次規格定稿**（無 UI）：客戶驗證完成，無待決回饋

### 3.3 交付開發包（交付給工程團隊）

| 項目 | 說明 | 範本 |
| :--- | :--- | :--- |
| **產出時機** | PRD 確認 + 雛型／規格定稿後 | — |
| **內容** | 精簡、僅本階段開發範圍；避免納入 Discovery、歷史分析 | [交付包結構範本](standards/交付包結構範本.md) |
| **放置** | 可為需求 Repo 之子目錄、壓縮包、或 Tag 對應版本 | — |
| **範本索引** | 見 [§6 範本與 AI 產出指引](#6-範本與-ai-產出指引) | `00_Templates/` |

## 4. 交付開發包結構

### 4.1 有 UI 之專案

```
交付開發包/
├── README.md                      # 交付說明、版次、對應 Tag
├── .cursorrules                   # 專案專用（公司標準精華 + 本專案客製）
├── 01_規格/
│   ├── PRD_本階段範圍摘要.md      # 本階段功能摘要
│   ├── 程式代號與權限矩陣.md      # 模組與主程式代號、權限矩陣
│   ├── 欄位名稱對照表.md         # 本階段涉及之欄位
│   ├── 情境與驗收說明.md         # 情境說明、驗收條件
│   └── 開發要求與資安清單.md     # 資安與技術要求精華
├── 02_雛型/
│   ├── prototype/                 # 本階段雛型 HTML（定稿版）
│   └── 操作步驟說明.md
└── 03_附錄/                       # 必要時才放
    └── 流程圖與架構.md
```

### 4.2 無 UI 之批次程式

```
交付開發包/
├── README.md
├── .cursorrules
├── 01_規格/
│   ├── PRD_本階段範圍摘要.md
│   ├── 程式代號與權限矩陣.md
│   ├── 欄位名稱對照表.md
│   ├── 批次規格/                  # 取代雛型
│   │   ├── BATCH001_每日匯出.md
│   │   └── ...
│   ├── 情境與驗收說明.md
│   └── 開發要求與資安清單.md
└── 02_附錄/
    └── 流程圖與架構.md
```

## 5. 迭代與變更流程

### 5.1 迭代觸發

- 客戶回饋
- 需求變更
- 上線後營運調整

### 5.2 迭代步驟

1. **需求方**整理為「需求變更單」（結構化格式，見 [Tpl_需求變更單](../../00_Templates/Tpl_需求變更單.md)）
2. **需求方**更新 PRD、雛型／批次規格、欄位對照表（在需求 Repo）
3. **需求方**產出「變更補充包」或「新階段交付包」
4. **需求方**通知工程團隊（版次、變更摘要、對應文件）
5. **工程團隊**於自有專案實作、差異分析
6. **每週同步**對齊進度與阻塞項

### 5.3 變更交付方式

| 變更規模 | 交付方式 |
| :--- | :--- |
| **小範圍** | 變更補充包：需求變更單 + 受影響規格 + 更新雛型 |
| **大範圍／新階段** | 完整重打包：重新產出完整交付開發包 |

## 6. 驗收條件

### 6.1 放置位置

| 位置 | 用途 |
| :--- | :--- |
| **需求 Repo** | `02_Analysis/YYYY-MM-DD_驗收條件_模組名.md` 或併入 PRD 章節 |
| **交付開發包** | `01_規格/情境與驗收說明.md` 內含驗收清單，或獨立 `驗收條件_本階段.md` |

### 6.2 格式與類型

- **類型代碼**：F（功能）、P（權限）、S（資安）、D（資料／欄位）、B（批次）
- **必備欄位**：ID、類型、描述、檢核方式、對應規格
- **撰寫原則**：每條可獨立判斷通過／不通過；避免模糊用語
- **範本**：見 [Tpl_驗收條件](../../00_Templates/Tpl_驗收條件.md)、[驗收條件範本](standards/驗收條件範本.md)

### 6.3 AI 檢核

驗收條件撰寫時，**檢核方式**可標註：操作測試、程式檢視、輸出檢查、自動化，供 AI 或人工依類型執行檢核。

## 8. 範本與 AI 產出指引

### 8.1 範本一覽

| 範本 | 用途 | 路徑 |
| :--- | :--- | :--- |
| 交付開發包 README | 交付說明、版次、對應 Tag | `Tpl_交付開發包_README.md` |
| PRD 本階段範圍摘要 | 本階段功能摘要（非全文） | `Tpl_PRD_本階段範圍摘要.md` |
| 程式代號與權限矩陣（交付用） | 模組與主程式、權限矩陣 | `Tpl_程式代號與權限矩陣_交付用.md` |
| 欄位名稱對照表（交付用） | 本階段涉及欄位 | `Tpl_欄位名稱對照表_交付用.md` |
| 情境與驗收說明（交付用） | 情境 + 驗收清單 | `Tpl_情境與驗收說明_交付用.md` |
| 開發要求與資安清單 | 資安與技術要求精華 | `Tpl_開發要求與資安清單.md` |
| 需求變更單 | 迭代時結構化變更紀錄 | `Tpl_需求變更單.md` |
| 批次規格 | 無 UI 批次程式規格 | `Tpl_批次規格.md` |
| 驗收條件 | 結構化驗收清單 | `Tpl_驗收條件.md` |

### 8.2 AI 產出交付文件時之指引

當需求方或 AI 需產出交付開發包時，請依下列步驟：

1. **確認交付門檻**：PRD 已確認、雛型／批次規格已定稿
2. **選用範本**：依專案類型（有 UI／無 UI）選用對應範本
3. **萃取本階段範圍**：僅納入本階段要開發之程式、欄位、情境，不納入 Discovery、歷史分析
4. **填寫 README**：版次、對應需求 Repo 之 Tag／commit、本階段範圍摘要
5. **合併 .cursorrules**：公司標準精華（見 `standards/cursorrules_範本.md`）+ 本專案程式代號、欄位、雛型路徑
6. **產出驗收條件**：依 [Tpl_驗收條件](../../00_Templates/Tpl_驗收條件.md) 格式，每條可檢核、有對應規格
