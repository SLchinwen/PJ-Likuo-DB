# 交付開發包｜補足建議與版本對照

本文件依 [單支程式交付開發包｜實作足夠性檢視與建議](單支程式交付開發包_實作足夠性檢視與建議.md) 之 P1／P2 建議，整理**治理方式**與**生成方式**如何補上，並給出**建議與合適的版本**，供產出與審核交付包時對照。

---

## 1. 補足總覽（依 P1／P2）

| 優先 | 建議項 | 治理補足 | 生成補足 |
| :--- | :--- | :--- | :--- |
| **P1** | 依賴與介面摘要 | 交付包結構範本、AI 產出指引：README／01_規格 含依賴清單；03_附錄 可含依賴介面摘要 | 範本：README 含依賴區塊；新增「依賴介面摘要」產出步驟 |
| **P1** | 錯誤與邊界情境 | 欄位對照表／情境：必填、格式／長度、錯誤情境；驗收含 S、D 類型 | 範本：欄位對照表_交付用 增欄位；情境與驗收 強調錯誤／邊界 |
| **P1** | 補齊交付用範本 | 交付包結構範本、AI 產出指引：範本路徑對照表完整 | 新建：README、PRD 摘要、情境與驗收、子程式清單、工程師開工指引、欄位對照表、權限矩陣、開發要求與資安 |
| **P2** | 技術棧與認證引用 | 交付包結構範本：README 或 03_附錄 註明技術棧／認證引用 | 範本：README 含技術棧／認證區塊 |

### 3.2 產出步驟對應（AI 產出交付文件指引）

- **Step 2**：使用 Tpl_交付開發包_README，填寫時含依賴清單、技術棧／認證。
- **Step 4**：欄位對照表依 Tpl_欄位名稱對照表_交付用，欄位含必填／格式／錯誤情境（若有）；情境與驗收依 Tpl_情境與驗收說明_交付用，驗收表含 S、D 類型。
- **Step 4.6（新增）**：有依賴時產出 03_附錄/依賴介面摘要.md 或於 README 明確列出並連結。
- **Step 5**：工程師開工指引依 Tpl_工程師開工指引，含交付包接收者自檢。

## 5. 補足後產出流程簡述

1. **產出前**：確認 PRD、雛型／批次規格定稿；範本取用本 Template Repo 之 `templates/` 下對應檔（如 tpl_交付開發包_readme.md、tpl_工程師開工指引.md 等）；專案 Repo 複製後為 00_Templates/。
2. **README**：依 Tpl_交付開發包_README 填寫，**必含**依賴清單、技術棧／認證引用。
3. **01_規格**：欄位對照表含必填／格式／錯誤情境（若有）；情境與驗收含結構化驗收表（含 S、D 類型）。
4. **03_附錄**：有依賴時產出依賴介面摘要或於 README 列出並連結；工程師開工指引含交付包接收者自檢。
5. **接收者**：取得包後依開工指引「交付包接收者自檢」勾選再開工。
