# 程式編碼原則｜共用規範

本文件為**跨專案、跨模組**共用之程式編碼原則，適用於報關管理、不同專案及其他模組。各專案／模組依此規範訂定**模組代號**與**百位數對應**，並產出權限表用主程式清單；子編碼由程式設計師依本規範編列。

**細部格式約束、禁止事項、雛型／TABLE／API 對齊、程式文件生成檢查清單**見 [模組與程式代碼編碼原則](模組與程式代碼編碼原則.md)；兩份併用，格式與命名以模組與程式代碼編碼原則為準。

---

## 1. 目的與適用範圍

- **目的**：統一程式代號結構與權限表編列方式，便於授權管理、跨專案查找與溝通。
- **適用範圍**：所有採用本規範之專案與模組（如報關管理及其他專案）；各專案／模組須選定**模組代號**與**主作業分類**後，依下列原則編列主程式與子編碼。

---

## 2. 代號結構

| 組成 | 說明 | 範例 |
| :--- | :--- | :--- |
| **模組代號** | **建議 2～3 碼**英文大寫（與 [模組與程式代碼編碼原則](模組與程式代碼編碼原則.md) 一致）；若專案採 3～5 碼縮寫須於專案內註明並與該文件對齊；同一專案內不重複 | `CUS`、`ORD`、`EXPO` 等，由各專案訂定 |
| **功能代號** | 三位數，百位數對應主作業類別，十位個位對應功能類型 | `110`、`120` |
| **完整主程式代號** | `<模組代號><功能代號>`，用於**使用者授權權限表** | `CUS110`、`CMS210`、`EXPO100` |

---

## 3. 權限表主程式原則（每作業四支）

**僅下列四類**編列為主程式代號，登錄於使用者授權之權限表；對應選單／功能入口，供權限控管使用。

| 十位個位 | 功能類型 | 說明 |
| :--- | :--- | :--- |
| **x00** | 作業目錄 | 該類作業入口／選單 |
| **x10** | 案件建檔 | 案件新增與維護；其內流程步驟（如成單、審核、帳務、結案等）為**子作業**，不另編主程式代號 |
| **x20** | 案件查詢（含增修） | 案件查詢與增刪修；若有匯出、統計需求，以子編碼延伸 |
| **x30** | 案件歷史歸檔 | 歷史歸檔與歷史資料查詢 |

- 每一**主作業**（由專案定義，如出口、進口、轉運…或其他專案之作業分類）對應一組百位數，並編列上述四支主程式（x00、x10、x20、x30）。
- **匯出、統計**等為程式延伸，不納入權限表主程式，以**子編碼**編列。

---

## 4. 子編碼原則（不納入權限表）

下列項目**不**給予獨立主程式代號，由**程式設計師**依團隊規則以**子編碼**編列，不列入使用者授權權限表。

### 4.1 建檔內子作業

- 建檔程式內之流程步驟（例如：成單與編號、文件審核、帳務、運輸安排、聯絡單輸出、結案等）屬**建檔之子作業**。
- 編碼方式：`<主程式代號>_<識別>`，例：`CUS110_Order`、`CUS110_DocAudit`、`CUS110_Report01`。
- 識別命名由程式設計師依專案慣例訂定（英文簡稱、序號等）。

### 4.2 報表、開窗、查詢子畫面

- 作業內之報表、開窗、查詢子畫面等附屬功能，不另編主程式代號。
- 編碼方式：`<主程式代號>_<識別>`，例：`CUS110_Report01`、`CUS120_Query01`。

### 4.3 匯出、統計（程式延伸）

- 若有需要，匯出、統計為程式延伸，以子編碼編列。
- 編碼方式：`<主程式代號>_<識別>`，例：`CUS120_Export01`、`CUS120_Report01`。

### 4.4 批次程式（不掛選單）

- **適用**：系統批次程式不掛選單，由排程或手動觸發執行。
- **對應主程式**：若批次程式對應**查核、變更狀態、資料同步**等與特定主程式相關之作業，則依該主程式代號編列子程式。
- **編碼方式**：`<主程式代號>_batch_<功能>`，例：`CUS110_batch_update`（出口建檔狀態更新）、`CUS110_batch_verify`（出口資料查核）、`CUS120_batch_status`（查詢批次狀態同步）。
- **無對應主程式**：若批次為全域或跨主程式（如每日匯出、月結轉檔），可採獨立編碼（如 `BATCH001`、`SYS_Batch_DailyExport`），由專案訂定。

### 4.5 子程式識別命名慣例

- **識別字元**：建議用英文字母、數字、底線；避免特殊字元。
- **大小寫**：建議統一採**小寫**，例：`CUS110_doc_audit`、`CUS110_report01`；`_batch_` 固定小寫。
- **批次識別**：`_batch_` 為固定關鍵字，其後功能採小寫，例：`CUS110_batch_update`、`CUS110_batch_verify`。
- **專案慣例**：若專案已採用 camelCase（如 `CUS110_Order`），可沿用，但須於專案架構文件註明。

### 4.6 全域批次編號規則

- **適用**：無對應主程式之批次（全域、跨主程式、系統級）。
- **格式**：`BATCH`＋三位數，或 `SYS_Batch_`＋功能英文；由專案於架構文件訂定。
- **範例**：`BATCH001`（每日匯出）、`BATCH002`（月結轉檔）、`SYS_Batch_DailyExport`。
- **編號**：`BATCH001`～`BATCH999` 依新增順序遞增；作廢保留編號，不重複使用。

### 4.7 編列權責

- **主程式代號**（權限表用）：由**需求方／專案**訂定，寫入各專案之「模組與程式代號架構」文件。
- **子編碼**（含批次）：由**程式設計師**依本規範與團隊規則編列，不列入權限表清單；建議於專案內留存子編碼清單或命名規則說明。

## 5. 系統人員專用程式編碼原則（CUSSYS）

**適用**：供**系統人員**使用之程式（如表單組態維護、參數設定、主檔維護等），與一般業務承辦使用之主程式區隔。

| 組成 | 說明 | 範例 |
| :--- | :--- | :--- |
| **模組代號** | 與主程式相同，維持模組一致性 | `CUS` |
| **SYS 標識** | 固定字串，表示「系統設定／系統人員專用」 | `SYS` |
| **三位數** | 依新增順序編列，010、020、030… | `010` |
| **完整代號** | `<模組代號>SYS<三位數>`，用於權限表 | `CUSSYS010` |

**多模組區別**：不同模組之系統程式各自編列，例：報關管理 `CUSSYS010`、若另有進口模組則 `IMPSYS010`，可作多模組時程式代號區別。

**權限**：僅授權予**系統管理者**或**系統人員**角色；一般承辦選單不顯示。

---

## 6. API 與 TABLE 命名對應

### 6.1 API 命名

- **對應**：API 路徑或端點名稱建議與程式代號對應，便於追溯。
- **格式**：由專案訂定；範例：`/api/<模組小寫>/<主程式>/<動作>`，如 `/api/cus/110/cases`、`/api/cus/120/query`。
- **批次 API**：若批次提供觸發或查詢 API，路徑可含子程式識別，如 `/api/cus/110/batch/update`。

### 6.2 TABLE／資料庫命名

- **對應**：資料表命名建議與模組、主程式或業務實體對應。
- **格式**：由專案訂定；範例：`<模組小寫>_<實體>`，如 `cus_export_master`、`cus_customer`、`cus110_cases`。
- **欄位**：依模組欄位名稱對照表之中英文對照命名；範本見 `../templates/tpl_欄位名稱對照表_交付用.md`（本 Template Repo）。

## 7. 各專案／模組之百位數規劃

- **百位數**由各專案／模組自訂，對應該專案之**主作業分類**（例如報關管理：1xx 出口、2xx 進口…；不同專案依其作業分類訂定）。
- 同一專案內，同一功能的十位個位意義需一致（x00 皆為目錄、x10 皆為建檔、x20 皆為查詢含增修、x30 皆為歷史歸檔）。
- 新增主作業時，以未使用之百位數遞增編列。

---

## 8. 使用原則（共通）

- 同一專案／模組必須使用同一**模組代號**。
- 權限表僅登錄**主程式代號**（x00、x10、x20、x30）；子作業與附屬功能不登錄。
- 文件、雛型、程式命名與程式代號保持一致；雛型 HTML 檔名建議：`YYYY-MM-DD_prototype_<程式代號>.html`（僅主程式代號時）。
- 作廢／刪除規則：若對外有編號，建議採作廢保留編號、系統單號重取，避免空號。

---

## 9. 專案套用方式

1. **選定模組代號**：依**系統名稱或主功能**之英文單字首組成，**三～五碼**；如報關管理 `CUS`（Customs）、`CMS`（Customs Management System），或各專案自訂（由貴單位訂定）。
2. **訂定主作業分類與百位數**：列出該模組之所有主作業，並分配百位數（1xx、2xx、3xx…）。
3. **產出主程式代號清單**：每一主作業編列四支主程式（目錄、建檔、查詢含增修、歷史歸檔），寫入該專案之架構或 PRD 文件。
4. **權限表**：僅登錄上述主程式代號。
5. **子編碼**：由程式設計師於開發時依本規範 §4 編列，並於專案內紀錄。
6. **API／TABLE**：依本規範 §5 或專案技術規格訂定命名對應。

---

## 10. 模組代號命名指引

| 項目 | 說明 |
| :--- | :--- |
| **長度** | 三～五碼（3～5 個英文字母） |
| **組成** | 由**系統名稱**或**主功能**之英文單字首組成 |
| **範例** | 報關管理 Customs → `CUS`；Customs Management System → `CMS`；出口作業 Export Operations → `EXPO`；電子發票 E-Invoice → `EINV` |
| **注意** | 同一專案／模組內不重複；避免與常見縮寫衝突 |

---

## 11. 範例摘要（報關管理模組）

| 項目 | 內容 |
| :--- | :--- |
| 模組代號 | `CUS`（Customs 報關，3 碼） |
| 主作業與百位數 | 1xx 出口、2xx 進口、3xx 轉運、4xx 機場、5xx SP、6xx 船用品 |
| 主程式範例 | CUS100 目錄、CUS110 建檔、CUS120 查詢（含增修）、CUS130 歷史歸檔（出口）；進口 CUS200～CUS230…依此類推 |
| 系統人員程式 | CUSSYS010 表單組態維護（模組代號＋SYS＋三位數，供多模組區別） |
| 子編碼範例 | 建檔內子作業：CUS110_Order、CUS110_Report01；查詢延伸：CUS120_Export01；批次（對應主程式）：CUS110_batch_update、CUS110_batch_verify |

不同專案或其它模組：依同原則訂定**模組代號**（建議 2～3 碼英文大寫，與模組與程式代碼編碼原則一致）與**百位數對應**後，套用 §3 之主程式結構即可。
