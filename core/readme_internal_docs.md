# 公司內部文件治理辦法與規則｜總覽

本 Template Repo 之治理入口為 **core/**（本文件即 core 之總覽）；以下說明適用於「複製至專案 Repo 後之 internal_docs/」。本目錄為**公司級內部文件治理**之中央庫，依 [公司專案標準規範｜需求與開發協作](公司專案標準規範_需求與開發協作.md) 建立結構。**各專案均可引用**本 repo 之治理辦法與規則，新專案可複製所需文件至該專案 `internal_docs/`（專案 Repo 內），或於交付包中註明引用版次。

- **對外**：對客戶之文件與雛型連結置於 `docs/`，由 MkDocs 建置，本目錄**不納入**對外發布。
- **對內**：本目錄及 `standards/` 下檔案均供開發團隊與專案管理使用，與 `公司專案標準規範` 無衝突，互為補充。

---

## 1. 文件結構總覽（依公司專案標準規範模式）

本 Template Repo 結構：**core/**（本總覽）、**standards/**、**templates/**、**ops/**、**index/**、**_archive/**。專案 Repo 複製後可採用以下結構：

```
internal_docs/                   ← 專案 Repo 內
├── README.md                    ← 本總覽（入口）
├── index.md                     ← 詳細一覽與分類索引
├── standards/                   ← 中央標準（交付包、PRD 拆分、驗收、資安等）
│   └── 00_總覽.md               ← 中央標準索引
└── [以下依類別分列於 index.md]
```

## 3. 各專案如何引用

**新專案引用時：複製即可。** 無需 git submodule；將本 repo 之所需目錄與檔案複製至新專案後，依專案名稱、Repo 網址、角色等替換即可使用。

| 情境 | 建議 |
| :--- | :--- |
| **新專案** | 依 [新專案參考使用辦法｜整合 AI 開發](新專案參考使用辦法_整合AI開發.md) 建立目錄與必備文件；**複製** [公司專案標準規範](公司專案標準規範_需求與開發協作.md)、[協作與治理規範](協作與治理規範.md)、[文件治理規範](docs_治理規範.md)、[standards/00_總覽](standards/00_總覽.md) 及所需 standards 範本至新專案 `internal_docs/`。 |
| **舊專案加入治理** | 依 [舊專案加入治理｜操作指引](舊專案加入治理_操作指引.md) 與 AI 協助步驟執行。 |
| **產出交付開發包** | 依 [協同開發｜文件治理流程](協同開發_文件治理流程.md)、[AI 產出交付文件指引](AI_產出交付文件指引.md) 及 [standards/交付包結構範本](standards/交付包結構範本.md) 產出。 |
| **開發團隊 AI 開發** | 依 [開發團隊 AI 開發指引](開發團隊_AI開發指引.md)、[模組與程式代碼編碼原則](模組與程式代碼編碼原則.md) 對齊命名與權限。 |

## 4. 與公司專案標準規範之對齊

本目錄所有文件與 [公司專案標準規範｜需求與開發協作](公司專案標準規範_需求與開發協作.md) **無衝突**，並補充下列細節：

- **協作與治理規範**、**docs_治理規範**：細化產物歸屬、PRD 四項必備、對外/對內規則、雛型資安。
- **standards/**：交付包、驗收、批次、資安、cursorrules 等為**公司級協同開發標準**，與程式編碼原則、文件治理 §6／§7 對應。
- **程式編碼原則**、**雛型產出規範**：與程式代號、權限表、雛型檔名及命名避免亂碼一致。
- **AI 相關**：AI 產出須經需求方審核、交付包與 Prompt 依中央標準產出，與整合 AI 開發檢核一致。
