# 中央標準｜工程師開工指引｜治理規範

本規範定義**各交付開發包**須含之**工程師開工指引**，供工程師與 AI 依此進行開發。指引置於交付包 `03_附錄/工程師開工指引.md`。

---

## 1. 適用時機

| 情境 | 適用 |
| :--- | :--- |
| **產出交付開發包** | 有 UI 或批次之主程式交付包，須含 `03_附錄/工程師開工指引.md` |
| **工程師開工** | 工程師取得交付包後，依指引進行準備與開發 |

---

## 2. 開工指引必備章節

| 章節 | 說明 |
| :--- | :--- |
| **1. 建議進行步驟** | 開工後依序執行之步驟（讀規格、建環境、產 schema、實作、驗收） |
| **2. AI Prompt 建議** | 工程師使用 Cursor／AI 時可複製之 Prompt 範例 |
| **3. 建議環境** | 本專案採用之雲端／技術棧；Azure 資源建議 |
| **4. 準備事項** | 開工前須完成之準備（帳號、權限、工具、依賴程式等） |
| **5. 交付包接收者自檢** | 取得交付包後、開工前勾選：依賴是否取得、schema_參考／API_規範是否在包內、驗收條件是否可執行、技術棧是否已知、環境與權限取得方式是否已知 |

---

## 3. 建議環境（本專案）

本專案採用 **Azure 雲端**：

| 服務 | 用途 | 說明 |
| :--- | :--- | :--- |
| **Azure SQL 彈性資料庫** | 資料庫 | 依欄位對照表與 schema 建立 TABLE |
| **App Service** | API 伺服器 | 部署後端 API |
| **Application Insights** | 監控 | 建議啟用以追蹤效能與錯誤 |
| **Azure Storage** | 檔案儲存 | 附件、匯出檔等（若需要） |
| **Log Analytics** | 日誌 | 集中日誌分析（若需要） |

公司現有 Azure 資源清單可參考專案 `internal_docs/Azure環境建議.md` 或由專案負責人提供。

---

## 4. 產出與檢查

產出交付包時，依 [Tpl_工程師開工指引](../../00_Templates/Tpl_工程師開工指引.md) 產出 `03_附錄/工程師開工指引.md`，並依本程式代號、依賴、雛型路徑客製化。

若開工指引尚未存在，AI 應**自動產出**並納入交付包。
