# 開發順序建議｜合群報關管理系統

本文件依 [TABLE 與 API 設計｜AI 產出指引](TABLE與API設計_AI產出指引.md) §3 產出，供專案排程與分工參考。

---

## 1. 原則

- **優先平行**：無依賴時可平行開發
- **依賴分組**：依賴性強時分為一組，組內依序、組間可平行

---

## 2. 建議順序

| 組別 | 程式 | 依賴 | 說明 |
| :--- | :--- | :--- | :--- |
| **A 組（可平行）** | CUS010 | — | 客戶主檔與聯絡人；基礎主檔 |
| **A 組** | CUSSYS010 | — | 表單組態維護；form_config |
| **B 組** | CUS110 | CUS010、CUSSYS010 | 出口建檔；需客戶開窗、form_config 選項 |
| **C 組** | CUS710 | CUS110（及未來進口等） | 運務查詢；資料來自建檔之運務需求 |

---

## 3. 依賴關係圖

```
A 組（可平行）：
  CUS010 ────────┐
                 ├──→ B 組：CUS110
  CUSSYS010 ─────┘

B 組完成後：
  CUS110（及未來進口等）──→ C 組：CUS710
```

---

## 4. 彈性調整

- **CUS110 與 CUSSYS010**：若 CUS110 可先以固定選項或 mock form_config 開發，則可部分平行
- **CUS710**：可於 CUS110 有運務資料後再開發，或先以 mock 資料開發 UI

---

## 5. 司機／廠商主檔

CUS710 指派需司機、廠商主檔。建議：

- **司機主檔**：可與 CUSSYS010 平行或於 CUS110 前建立；須支援整批匯出與匯入
- **供應商主檔**：同上；與出口建檔之「派外車供應商」可共用
