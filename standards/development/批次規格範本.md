# 中央標準｜批次規格範本

本文件定義**無 UI 批次程式**之規格格式，取代雛型作為交付依據。每支批次一檔。

---

## 1. 檔案命名與程式代號

### 1.1 程式代號選用

| 情境 | 程式代號格式 | 範例 |
| :--- | :--- | :--- |
| **對應主程式**（查核、變更狀態、資料同步） | `<主程式代號>_batch_<功能>` | `CUS110_batch_update`、`CUS110_batch_verify` |
| **無對應主程式**（全域、跨主程式） | `BATCH`＋三位數 或 `SYS_Batch_`＋功能 | `BATCH001`、`BATCH002`、`SYS_Batch_DailyExport` |

見 [程式編碼原則｜共用規範](../../程式編碼原則_共用規範.md) §4.4 批次程式、§4.6 全域批次編號規則。

### 1.2 規格檔命名

格式：`<程式代號>_<簡述>.md`

範例：`CUS110_batch_update_狀態同步.md`、`BATCH001_每日匯出.md`

---

## 2. 規格章節結構

```markdown
## [程式代號]｜[批次名稱]

### 觸發條件
- 排程：每日 02:00
- 前置：前日結關資料已就緒

### 輸入
| 來源 | 格式 | 說明 |
|------|------|------|
| 報關主檔 | TABLE: decl_master | 結關日 = 前一日 |
| 客戶主檔 | TABLE: customer | 依統編關聯 |

### 處理邏輯
1. 篩選結關日 = T-1 之報關單
2. 依客戶統編關聯客戶主檔
3. 產出 CSV，欄位依「欄位名稱對照表」

### 輸出
| 項目 | 規格 |
|------|------|
| 檔名 | export_YYYYMMDD_HHmmss.csv |
| 路徑 | /output/daily/ |
| 欄位順序 | 依欄位對照表 §X |

### 錯誤處理
- 無資料：產出空檔，寫入日誌
- 主檔缺漏：跳過該筆，寫入錯誤檔 error_YYYYMMDD.csv

### 驗收條件
| ID | 類型 | 描述 | 檢核方式 |
|----|------|------|----------|
| BATCH001-B01 | B | 產出檔名格式 export_YYYYMMDD_HHmmss.csv | 執行後檢查檔名 |
| BATCH001-B02 | B | 無資料時產出空檔並寫入日誌 | 模擬無資料執行 |
```

---

## 3. 各章節說明

| 章節 | 必備 | 說明 |
| :--- | :--- | :--- |
| 觸發條件 | ✅ | 排程時間、手動觸發、前置條件 |
| 輸入 | ✅ | 資料來源、格式、篩選條件 |
| 處理邏輯 | ✅ | 步驟化描述，必要時附流程圖 |
| 輸出 | ✅ | 檔名、路徑、格式、欄位順序 |
| 錯誤處理 | ✅ | 異常情境與對應行為 |
| 驗收條件 | ✅ | 至少 1 條 B 類型，見 [驗收條件範本](驗收條件範本.md) |

---

## 4. 與欄位對照表之對應

- 輸出欄位須對應「欄位名稱對照表」之中英文、縮寫
- 若批次涉及特殊欄位，可於本規格內補充，並提醒更新對照表
