# 中央標準｜cursorrules 範本

本文件為**公司級 AI 指令範本**，供產出交付開發包之 `.cursorrules` 時使用。交付時須將**公司標準精華**與**本專案客製**合併寫入。

---

## 1. 使用方式

產出交付包時，複製以下內容至 `交付開發包/.cursorrules`，並替換 `[專案客製]` 區塊。

---

## 2. 範本內容

```markdown
# Role (角色設定)
你是協助本專案開發的 AI，依交付包內之規格、雛型／批次規格與驗收條件進行實作與檢核。

# 語言與格式
- 預設使用「臺灣繁體中文」
- 文件輸出為 Markdown (.md)
- 關鍵字使用 **粗體** 強調

# 本專案規格依據（專案客製）
[專案客製] 請填入：
- 本階段程式代號清單：CUS100、CUS110、...
- 雛型路徑：01_規格/、02_雛型/prototype/
- 欄位對照表路徑：01_規格/欄位名稱對照表.md
- 驗收條件路徑：01_規格/情境與驗收說明.md
- 子程式清單路徑：01_規格/子程式清單.md（實作時依此展開工作包；見 internal_docs/standards/子程式實作建議_標準.md）

# 命名與編碼
- 程式代號、TABLE、API 命名依「程式代號與權限矩陣」「欄位名稱對照表」
- 子程式識別採小寫；批次格式 `<主程式>_batch_<功能>`；全域批次 `BATCH001` 等；見 01_規格 或程式編碼原則
- 雛型檔名：YYYY-MM-DD_prototype_<程式代號>.html

# 權限與資安
- 依權限矩陣表實作，採最小權限原則
- 前後端皆須驗證身分與權限
- 密碼不明文儲存；敏感資料遮罩；HTTPS 傳輸
- 詳見 01_規格/開發要求與資安清單.md

# 驗收
- 實作完成後依「情境與驗收說明」內之驗收條件自檢
- 每條驗收條件可獨立判斷通過／不通過
```

---

## 3. 專案客製填寫指引

| 項目 | 填寫範例 |
| :--- | :--- |
| 程式代號清單 | CUS100、CUS110、CUS120、BATCH001 |
| 雛型路徑 | 02_雛型/prototype/ |
| 欄位對照表 | 01_規格/欄位名稱對照表.md |
| 驗收條件 | 01_規格/情境與驗收說明.md |
| 子程式清單 | 01_規格/子程式清單.md（有 UI 必備；實作時依此展開） |
| 批次規格路徑 | 01_規格/批次規格/（無 UI 專案） |
