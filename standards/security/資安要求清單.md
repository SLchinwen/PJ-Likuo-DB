# 中央標準｜資安要求清單

本文件萃取 [協作與治理規範](../協作與治理規範.md) §2.5 與 [文件治理規範](../docs_治理規範.md) §7，供交付包、PRD、雛型產出時引用。開發方規劃與實作時須依公司或客戶既有資安規範從嚴辦理。

---

## 1. 開發階段（開發方規劃與實作時）

| 類別 | 提醒要點 |
| :--- | :--- |
| **身分認證與登入** | 密碼不可明文儲存；登入／Session 逾時與登出機制；若客戶或公司要求，考量多因素認證（MFA） |
| **授權與權限** | 依 PRD 之**權限矩陣表**與情境實作，採**最小權限**原則；前後端皆須驗證身分與權限，不得僅依前端隱藏按鈕 |
| **敏感資料** | 報關資料、個資、客戶資料之存取與儲存須受控；畫面與報表依角色遮罩或限制顯示；日誌與除錯輸出**不得**含密碼、完整個資或敏感欄位 |
| **傳輸** | 對外或敏感連線使用 **HTTPS**；避免將敏感參數置於 URL 或 Cookie；API 與前後端通訊依環境要求加密 |
| **日誌與稽核** | 重要操作（登入、權限變更、關鍵業務異動）可追溯；日誌內容避免記錄敏感資料；依公司或客戶規定保留與保護 |
| **依循既有規範** | 依**公司資安政策**與**客戶資安／個資要求**（若有）規劃；若有弱掃、滲透測試或合規需求，預留時程與項目 |

---

## 2. 雛型設計階段（需求方或產出雛型時）

| 對應類別 | 雛型設計／生成時建議 |
| :--- | :--- |
| **授權與權限** | 雛型中區分角色示範（標示「僅 XX 角色可見」、選單／功能差異） |
| **敏感資料** | 敏感欄位以遮罩或假資料示範，不使用真實客戶／報關資料 |
| **登入與登出** | 保留登出示意；可註記逾時與 Session 由正式系統實作 |
| **傳輸與環境** | 註記「正式環境須 HTTPS」「實際送出須經後端驗證與加密」 |
| **對齊整體要求** | 參照協作與治理 §2.5；客戶／公司額外資安規定寫入 PRD 或雛型說明 |

---

## 3. 交付包內之呈現

產出「開發要求與資安清單」時，可依本清單精簡為 1 頁，並補充本專案之額外資安或合規需求。
