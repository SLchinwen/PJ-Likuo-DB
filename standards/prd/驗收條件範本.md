# 中央標準｜驗收條件範本

本文件定義**驗收條件**之格式、類型與撰寫原則，供 AI 與人工產出與檢核時依循。

---

## 1. 類型代碼

| 代碼 | 類型 | 說明 | 檢核方式範例 |
| :--- | :--- | :--- | :--- |
| F | 功能 | 功能行為、操作流程 | 操作測試、自動化測試 |
| P | 權限 | 角色與程式權限對應 | 操作測試（不同角色登入） |
| S | 資安 | 密碼、傳輸、敏感資料 | 程式檢視、弱掃 |
| D | 資料／欄位 | 欄位命名、儲存、輸出格式 | 程式檢視、輸出檢查 |
| B | 批次 | 批次輸入、輸出、錯誤處理 | 執行測試、輸出檢查 |

---

## 2. 必備欄位

每條驗收條件須含：

| 欄位 | 說明 |
| :--- | :--- |
| **ID** | 唯一識別，建議格式：`<程式代號>-<類型><序號>`，例：CUS100-F01、BATCH001-B01 |
| **類型** | F／P／S／D／B |
| **描述** | 一句話、可檢核之描述；避免「應合理」「應正常」等模糊用語 |
| **檢核方式** | 操作測試／程式檢視／輸出檢查／自動化 |
| **對應規格** | PRD 章節、客戶回饋進版、批次規格等，便於追溯 |

---

## 3. 表格格式範例

```markdown
| ID | 類型 | 描述 | 檢核方式 | 對應規格 |
|----|------|------|----------|----------|
| CUS100-F01 | F | Step1 客戶欄位：輸入客戶可帶入統編 | 操作測試：選客戶後統編自動帶入 | PRD §4.1 |
| CUS100-F02 | F | S/O 僅能輸入 4 碼 | 輸入 5 碼應被拒絕 | 客戶回饋進版 |
| CUS100-P01 | P | 承辦角色可新增案件 | 以承辦帳號登入可看到新增按鈕 | 權限矩陣 |
| CUS100-S01 | S | 密碼不明文儲存 | 檢查 DB／日誌無明文密碼 | 資安清單 |
| CUS100-D01 | D | 船名／航次分兩欄儲存，聯絡單合併顯示 | 查 DB 兩欄；列印單一欄 | PRD §4.2 |
| BATCH001-B01 | B | 產出檔名格式 export_YYYYMMDD_HHmmss.csv | 執行後檢查檔名 | 批次規格 |
```

---

## 4. 撰寫原則

### 4.1 可檢核

| ✅ 良好 | ❌ 避免 |
| :--- | :--- |
| 「S/O 欄位僅接受 4 位數字，輸入 5 位時顯示錯誤」 | 「系統應正常運作」 |
| 「權限矩陣：承辦可執行 CUS110；以承辦帳號登入應可見 CUS110」 | 「權限應正確」 |
| 「欄位對照表：船名→vessel_name；DB 欄位需為 vessel_name」 | 「命名應一致」 |

### 4.2 批次程式

- 至少涵蓋：輸入來源、輸出格式、錯誤處理
- 每支批次至少 1 條 B 類型驗收（輸出格式或執行結果）

### 4.3 放置位置

- **需求 Repo**：`02_Analysis/YYYY-MM-DD_驗收條件_模組名.md` 或併入 PRD
- **交付開發包**：`01_規格/情境與驗收說明.md` 或獨立 `驗收條件_本階段.md`

---

## 5. AI 檢核指引

當 AI 協助檢核驗收時：

1. 依 **檢核方式** 執行：操作測試需人工；程式檢視、輸出檢查可依規格比對
2. 依 **對應規格** 取得預期結果，比對實際產出
3. 每條產出：通過／不通過／待確認，並註記檢核依據
