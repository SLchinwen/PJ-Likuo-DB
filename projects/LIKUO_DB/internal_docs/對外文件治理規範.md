# 立國工程專案｜對外文件治理規範

- **日期：** 2026-03-03
- **版本：** v1.1
- **用途：** 定義立國工程專案（MCR、HR）對外文件的目錄結構、命名規範與發布原則。

---

## 1. 對外文件（docs/）定位

**用途**：供**使用者**了解系統目標、應用說明與雛型預覽。

**放置位置**：`docs/`，由 MkDocs 建置並發布至 GitHub Pages。

**納入範圍**：

- 系統整體目標、應用說明、發展目標
- 程式清單（程式代號、名稱、用途、雛型檢視連結）
- 雛型操作步驟說明
- 雛型 HTML（`prototype_<程式代號>.html`）

---

## 2. 內部文件（不納入對外公開）

**用途**：供**開發團隊**協作、交付與治理。

**放置位置**：`internal_docs/`、`90_delivery_packages/`、`01_Discovery/`、`02_Analysis/`、`00_Templates/` 等。

**不納入對外公開之內容**：

- **交付開發包**：目錄結構、交付包清單、內部路徑
- PRD 全文、規格書、工程師開工指引
- 訪談筆記、分析文件、內部協作流程

---

## 3. 對外文件治理原則

| 原則 | 說明 |
| :--- | :--- |
| **使用者導向** | 對外內容以**使用者**能理解與操作為主，不含內部開發術語與路徑。 |
| **不納入交付開發包** | 交付開發包為**內部開發交付文件**，不對外公開。 |
| **雛型連結穩定** | 對外雛型採 `prototype_<程式代號>.html` 命名，**去除日期前綴**，確保使用者取得最新版且連結不變。 |
| **導航層級扁平化** | 導航目錄不超過 **2 階**，善用頁面內連結與卡片式導航，提升專案網站美觀與易用性。 |

---

## 4. 對外文件目錄結構（MkDocs）

依軟體工程階段排序，適用於 MkDocs Material 主題。

### 4.1 導航結構（mkdocs.yml nav）

```yaml
nav:
  - 首頁: index.md
  - 系統範圍與規劃: scope.md
  - 功能與雛型預覽: features.md
  - 操作手冊:
      - 總覽: guides/index.md
      - 第一階段操作手冊（MCR）: guides/mcr_phase1.md
      - MCR310 客戶合約工作項目建檔: guides/prototype_MCR310.md
      - MCR320 客戶設備工作項目建檔: guides/prototype_MCR320.md
  - 版本與更新紀錄: changelog.md
```

### 4.2 實體目錄結構

```
projects/LIKUO_DB/docs/
├── index.md                                  # ❶ 首頁（Discovery）
├── scope.md                                  # ❷ 系統範圍與規劃（Planning）
├── features.md                               # ❸ 功能與雛型預覽（Design）
├── guides/                                   # ❹ 操作手冊（Delivery）
│   ├── index.md                              #    手冊總覽
│   ├── mcr_phase1.md                         #    第一階段操作手冊
│   ├── prototype_MCR310.md                   #    MCR310 操作說明
│   └── prototype_MCR320.md                   #    MCR320 操作說明
├── prototype/                                #    雛型靜態資源（不出現在導航）
│   ├── prototype_MCR310.html                 #    去除日期前綴
│   ├── prototype_MCR310_匯入範本.csv         #    去除日期前綴
│   ├── prototype_MCR320.html
│   ├── prototype_MCR320_匯入範本.csv
│   └── prototype_CraneDashboard.html
└── changelog.md                              # ❺ 版本與更新紀錄（Maintenance）
```

### 4.3 版面與樣式規範

- **導航圖示**：避免使用 MkDocs Material 預設的狀態圖示（如驚嘆號）。
  - **YAML 欄位調整**：將 `status` 欄位更名為 `doc_status`，避免觸發主題內建狀態功能。
- **目錄字體**：右側目錄（Table of Contents）字體應縮小一號（約 .65rem），以利閱讀。
  - 實作：透過 `docs/stylesheets/extra.css` 覆寫 `.md-sidebar--secondary .md-nav__link`。
- **頁尾版權**：顯示最後生成時間與維護者。
  - 格式：`Made with Material for MkDocs [Last Update: YYYY-MM-DD HH:MM] by SystemLead`
  - 實作：由 CI/CD 流程（GitHub Actions）在建置時動態注入 `mkdocs.yml` 的 `copyright` 欄位。

## 5. 軟工階段對照表

| 軟工階段 | 對外頁面 | 導航層級 | 頁面內連結 |
|----------|---------|---------|-----------|
| Discovery（探索） | `index.md` 專案目標 | 第 1 層 | 快速導航卡片 |
| Planning（規劃） | `scope.md` 範圍與階段 | 第 1 層 | Mermaid 藍圖 |
| Design（設計） | `features.md` 程式清單+雛型 | 第 1 層 | 雛型 HTML 連結、操作說明連結 |
| Delivery（交付） | `guides/*` 操作手冊 | 第 2 層 | 各手冊頁面 |
| Maintenance（維運） | `changelog.md` 版本紀錄 | 第 1 層 | 時間軸式列表 |
