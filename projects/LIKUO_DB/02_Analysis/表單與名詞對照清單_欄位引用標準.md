# 表單與名詞對照清單｜欄位引用標準

- **日期：** 2026-02-10
- **用途：** 統一專案內各文件、程式與表單之名詞定義與欄位命名標準，確保 PRD、規格、交付包與 UI 一致

---

## 1. 適用範圍

本文件適用於 MCR 系統所有模組（第一階段＋第二階段），包含：
- PRD / 需求文件中的名詞
- 資料庫欄位命名
- API 參數命名
- UI 顯示標籤
- 報表欄位標題

---

## 2. 命名規則

### 2.1 資料庫欄位

- 使用 **snake_case**（小寫＋底線）
- 前綴依模組區分：無全域前綴，由 table 名區隔
- Boolean 欄位以 `is_` 或 `need_` 開頭
- 日期時間以 `_at` 或 `_date` 結尾
- 狀態欄位以 `_status` 結尾

### 2.2 API 參數

- 使用 **camelCase**
- 與資料庫欄位之對應需明確記錄於本文件

### 2.3 UI 標籤

- 使用**繁體中文**
- 標籤長度不超過 8 個字（超過者需縮寫並加 tooltip）

## 5. 欄位引用規則

### 5.1 文件中引用欄位

- 在 PRD 或規格文件中引用欄位時，使用 `code` 格式標示英文欄位名
- 首次出現需加中文說明，例如：`need_photo`（需要拍照）
- 後續引用可僅使用英文欄位名

### 5.2 跨模組引用

- 引用其他模組欄位時需標註來源表，例如：`process.crane_tonnage`
- 唯讀欄位需標註「唯讀」，並說明修改來源

### 5.3 版本管理

- 新增欄位需記錄於本文件並標註新增日期
- 欄位更名需保留舊名對照，至少保留一個版本週期
