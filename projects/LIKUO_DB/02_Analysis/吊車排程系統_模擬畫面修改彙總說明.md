# 吊車排程系統－模擬畫面修改彙總說明

- **日期：** 2026-02-18
- **用途：** 彙整吊車排程系統各模擬畫面的修改要點，作為雛型迭代與客戶確認依據
- **雛型參考：** [吊車儀錶板](../docs/prototype/吊車儀錶板.html)

---

## 1. 修改背景

第二階段新增吊車排程相關功能，包含噸數彙總、停駐點管理、派車決策看板。在雛型 review 過程中，客戶與內部陸續提出畫面調整需求，本文件彙整截至目前的修改項目與狀態。

## 3. 各畫面修改明細

### 3.1 UI-01：吊車彙總看板（主畫面）

**修改前問題：**
- 資訊密度過高，主任難以快速判斷
- Ready / Expected 狀態未明顯區分
- 噸數與停駐點未同時呈現

**修改內容：**
- 將看板拆分為上下兩區：上方為統計摘要卡片，下方為明細列表
- 統計卡片包含：今日待派車數、已派車數、可用吊車數、噸數分佈
- 明細列表支援依停駐點群組、噸數範圍、狀態篩選
- 狀態以色標區分：Ready（綠）、Expected（黃）、Unavailable（灰）

**客戶回饋：**
- ✅ 同意版面拆分方案
- 🔄 要求統計卡片可收合
- 🔄 建議增加「本週」彙總視圖

### 3.2 UI-02：停駐點群組篩選

**修改內容：**
- 新增停駐點群組下拉篩選器（支援多選）
- 篩選後看板與列表即時更新
- 支援「全部」與「未分組」選項

**備註：** 停駐點群組資料來源為工序主檔中的 parking_zone_group 欄位

### 3.3 UI-03：噸數分佈圖

**修改前問題：**
- 原為純文字列表，不直觀
- 無法快速看出噸數集中區間

**修改內容：**
- 改為水平長條圖，X 軸為吊車數量，Y 軸為噸數區間
- 區間劃分：≤25T、25–50T、50–100T、100–200T、>200T
- 點擊長條可展開該區間的吊車列表
- 支援 Ready / Expected 堆疊顯示

**進度：** 圖表元件已完成，資料對接中

### 3.4 UI-04：派車指派對話框

**修改內容：**
- 彈出對話框，顯示：工地名稱、需求噸數、預計使用時間、停駐點
- 可選擇吊車（依噸數與可用狀態自動篩選）
- 可指派司機（顯示司機當日排程）
- 確認後發送通知至現場人員與司機

**欄位：**

| 欄位 | 說明 | 必填 |
|---|---|---|
| 工地/案場 | 下拉選取 | ✅ |
| 需求噸數 | 數值輸入 | ✅ |
| 預計使用時段 | 日期＋時間範圍 | ✅ |
| 停駐點 | 依群組自動帶入 | ✅ |
| 指派吊車 | 可用清單選取 | ✅ |
| 指派司機 | 可用清單選取 | ✅ |
| 備註 | 文字輸入 | — |

### 3.5 UI-05：吊車行事曆檢視

**規劃說明：**
- 以月曆或週曆格式呈現各吊車排程
- 每格顯示吊車編號、工地、噸數
- 支援拖曳調整排程（需確認權限）
- 與主看板連動

**評估重點：** 行事曆套件選型（FullCalendar vs 自建）

### 3.6 UI-06：司機資訊卡片

**規劃說明：**
- 點擊司機名稱展開資訊卡片
- 顯示：姓名、聯絡方式、持有證照、當日排程、可用狀態
- 來源：司機主檔

### 3.7 UI-07：吊車使用記錄列表

**修改前問題：**
- 欄位過多，水平捲動不便
- 缺乏日期範圍篩選

**修改內容：**
- 精簡為核心欄位：日期、吊車編號、工地、噸數、司機、狀態
- 新增日期範圍篩選（預設近 30 天）
- 支援匯出 Excel
- 可展開列顯示完整工項明細

### 3.8 UI-08：停駐點地圖視圖

**評估說明：**
- 以地圖標註各停駐點位置與當前吊車分佈
- 需評估地圖服務（Google Maps / OpenStreetMap）授權
- 目前列為 Phase 3 候選功能

## 4. 共通設計原則

### 4.1 色彩規範

| 狀態 | 色碼 | 說明 |
|---|---|---|
| Ready | #4CAF50（綠） | 可用 |
| Expected | #FFC107（黃） | 預期到達 |
| In Use | #2196F3（藍） | 使用中 |
| Unavailable | #9E9E9E（灰） | 不可用 |
| Overdue | #F44336（紅） | 逾時未歸 |

### 4.2 響應式設計

- 主看板支援 1920×1080 以上桌面解析度
- 列表頁支援平板橫向（1024×768）
- 派車對話框在平板直向可正常操作
