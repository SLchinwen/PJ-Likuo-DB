# PRD｜出缺勤系統（人臉／指紋辨識）

- **日期：** 2026-02-15
- **專案 PM：** chinwen
- **版本：** v1.0 Draft
- **用途：** 定義出缺勤系統的產品需求，涵蓋人臉與指紋辨識打卡、班表管理、異常處理與報表產出

---

## 1. 產品概述

### 1.1 背景

立國工程現行出缺勤管理以紙本簽到或傳統打卡鐘為主，現場人員分散於多個廠區與工地，難以即時掌握人員到勤狀態。隨著工程規模擴大與合規要求提升，需建立數位化出缺勤管理系統，支援人臉辨識與指紋辨識雙模式打卡，降低代打卡風險並提升管理效率。

### 1.2 產品定位

本系統為立國工程內部人力管理平台之子系統，專責出缺勤記錄、班表管理與異常處理，並與薪資結算模組對接。

### 1.3 目標使用者

| 角色 | 說明 |
|---|---|
| 現場人員 | 工程師、技術員、現場作業員 |
| 工地主任 | 負責該工地人員調度與到勤管理 |
| 人資管理員 | 總部出缺勤管理與報表審核 |
| 系統管理員 | 裝置管理、人員生物特徵建檔 |

### 1.4 成功指標（KPI）

- 打卡即時率 ≥ 95%（打卡後 5 秒內上傳成功）
- 代打卡事件趨近 0
- 異常出勤處理時效 ≤ 1 個工作日
- 月報產出時間由 3 天縮短至即時

## 2. 系統範圍

### 2.1 In Scope（本階段）

- 人臉辨識打卡（含活體偵測）
- 指紋辨識打卡（備援模式）
- 班表管理（固定班／排班／彈性班）
- 打卡記錄即時上傳與查詢
- 異常出勤申請與審核（補打卡、請假、加班）
- 出缺勤日報／月報產出
- 與薪資模組對接之資料介面

### 2.2 Out of Scope（本階段不含）

- GPS 定位打卡（Phase 2 評估）
- 訪客管理
- 門禁聯動控制
- 行動裝置 App 打卡（Phase 2 評估）

## 3. 使用者情境

### SC-ATT-01：現場人員上班打卡

**角色：** 現場人員
**前置條件：** 人員已完成人臉／指紋建檔
**主要流程：**
1. 人員抵達工地，走向打卡裝置
2. 選擇人臉辨識或指紋辨識模式
3. 系統比對生物特徵，辨識成功後記錄打卡時間
4. 裝置顯示「打卡成功」與時間
5. 記錄即時上傳至後端系統

**替代流程：**
- 3a. 人臉辨識失敗 → 提示改用指紋辨識
- 3b. 指紋辨識失敗 → 提示聯繫工地主任手動登記
- 5a. 網路中斷 → 本地暫存，恢復後自動上傳

### SC-ATT-02：工地主任查看即時到勤

**角色：** 工地主任
**前置條件：** 當日班表已排定
**主要流程：**
1. 登入系統，進入「即時到勤看板」
2. 查看該工地當日應到人數、已到人數、未到人數
3. 點擊未到名單，檢視是否有請假或異常
4. 對未到且未請假者發送提醒通知

### SC-ATT-03：異常出勤申請（補打卡）

**角色：** 現場人員
**前置條件：** 當日有漏打卡紀錄
**主要流程：**
1. 人員登入系統，進入「異常申請」
2. 選擇「補打卡」，填寫日期、時間、原因
3. 附上佐證資料（選填）
4. 提交申請，系統通知工地主任審核
5. 工地主任核准／退回

### SC-ATT-04：月報產出

**角色：** 人資管理員
**前置條件：** 月份結束且異常處理已完結
**主要流程：**
1. 登入系統，進入「報表中心」
2. 選擇月份與範圍（全公司／特定工地）
3. 系統自動彙算出勤天數、遲到、早退、請假、加班
4. 產出 Excel／PDF 報表
5. 可匯出至薪資模組對接格式

### SC-ATT-05：人員生物特徵建檔

**角色：** 系統管理員
**前置條件：** 人員已完成人事資料建立
**主要流程：**
1. 系統管理員登入管理後台
2. 選擇待建檔人員，進入「生物特徵建檔」
3. 引導人員進行人臉拍攝（正面、左右各 15°）
4. 引導人員進行指紋登錄（左右食指各 3 次）
5. 系統驗證特徵品質，品質通過後儲存
6. 將特徵資料同步至各工地打卡裝置

## 4. 功能需求

### 4.1 人臉辨識模組

| 項目 | 規格 |
|---|---|
| 辨識方式 | 近紅外線（NIR）人臉辨識 |
| 活體偵測 | 必須，防止照片／影片攻擊 |
| 辨識時間 | ≤ 1 秒 |
| 辨識準確率 | ≥ 99.5%（FAR ≤ 0.001%） |
| 支援人數 | 單一裝置 ≥ 5,000 人 |
| 離線辨識 | 支援（本地比對） |
| 環境適應 | 室外逆光、低光源、配戴安全帽／口罩 |

### 4.2 指紋辨識模組

| 項目 | 規格 |
|---|---|
| 感應器類型 | 光學式或電容式 |
| 辨識時間 | ≤ 1 秒 |
| 辨識準確率 | ≥ 99.0% |
| 支援人數 | 單一裝置 ≥ 10,000 枚 |
| 防水防塵 | IP65 以上（工地環境） |

### 4.3 班表管理

- **班別定義：** 固定班、二班制、三班制、彈性班
- **排班規則：** 依工地／專案／角色排班
- **班表異動：** 換班申請、加班申請需審核
- **班表發布：** 提前 7 天發布，異動需通知相關人員
- **法規遵循：** 勞基法工時上限提醒、連續工作天數警示

### 4.4 打卡規則引擎

- **遲到定義：** 超過班次開始時間 N 分鐘（可設定）
- **早退定義：** 早於班次結束時間 M 分鐘（可設定）
- **彈性時間：** 可設定寬限期（例如前後 10 分鐘）
- **加班認定：** 超過班次結束時間且有加班申請
- **跨日班次：** 支援夜班跨日打卡

### 4.5 異常處理

| 異常類型 | 申請方式 | 審核者 | 備註 |
|---|---|---|---|
| 補打卡 | 線上申請 | 工地主任 | 需填原因 |
| 請假 | 線上申請 | 工地主任→人資 | 依假別區分 |
| 加班 | 線上申請 | 工地主任 | 需符合勞基法 |
| 出差 | 線上申請 | 主管 | 出差地點記錄 |
| 裝置異常 | 系統管理員處理 | — | 自動偵測通報 |

### 4.6 報表與統計

- **日報：** 各工地當日出勤統計（到勤率、遲到人數、異常數）
- **月報：** 個人出勤明細、工地彙總、全公司彙總
- **異常報表：** 補打卡次數排行、遲到頻次分析
- **匯出格式：** Excel（.xlsx）、PDF、CSV
- **薪資對接：** 依約定格式產出薪資計算用資料

## 5. 資料模型

### 5.1 核心實體

| 實體 | 說明 | 主要欄位 |
|---|---|---|
| Employee | 人員主檔 | emp_id, name, dept, site_id |
| BiometricData | 生物特徵 | emp_id, face_template, fingerprint_template, status |
| Schedule | 班表 | schedule_id, emp_id, shift_type, date, start_time, end_time |
| PunchRecord | 打卡記錄 | record_id, emp_id, punch_time, punch_type (in/out), method (face/finger), device_id |
| AbnormalRequest | 異常申請 | request_id, emp_id, type, reason, status, approver_id |
| Device | 打卡裝置 | device_id, site_id, location, status, last_sync |

### 5.2 狀態機

**打卡記錄狀態：**
- Normal → 正常打卡
- Late → 遲到
- EarlyLeave → 早退
- Absent → 缺勤（未打卡且無請假）
- Adjusted → 已補登（經核准）

**異常申請狀態：**
- Draft → Submitted → Approved / Rejected

## 6. 介面需求

### 6.1 打卡裝置端

- 7 吋以上觸控螢幕
- 顯示時間、辨識結果、人員姓名與照片
- 支援中文介面
- 支援離線模式指示燈

### 6.2 Web 管理端

- **到勤看板：** 即時顯示各工地到勤狀態
- **班表管理：** 拖曳式排班介面
- **異常處理：** 待審核清單與一鍵核准
- **報表中心：** 條件篩選與即時預覽
- **裝置管理：** 裝置狀態監控與遠端設定

### 6.3 通知機制

- 未到勤提醒（推播／簡訊）
- 異常申請審核結果通知
- 班表異動通知
- 裝置離線告警

## 7. 非功能需求

| 項目 | 規格 |
|---|---|
| 可用性 | 99.5%（裝置端含離線模式） |
| 效能 | 打卡辨識 ≤ 1 秒、報表產出 ≤ 30 秒 |
| 安全性 | 生物特徵資料加密儲存、傳輸 TLS 1.2+ |
| 隱私 | 符合個資法，特徵資料不可還原為原始影像 |
| 容量 | 支援 500+ 同時在線裝置、10,000+ 人員 |
| 備援 | 裝置離線暫存 ≥ 7 天資料 |

## 8. 硬體需求

### 8.1 打卡裝置

| 項目 | 規格 |
|---|---|
| 處理器 | ARM Cortex-A53 以上 |
| 記憶體 | ≥ 2 GB |
| 儲存 | ≥ 8 GB（離線暫存） |

- 資料庫：SQL Server / PostgreSQL
