# PRD｜立國工程 MCR 第二階段

- **日期：** 2026-02-10
- **專案 PM：** chinwen
- **版本：** v1.0 Draft
- **用途：** 定義 MCR 第二階段產品需求，涵蓋回報/拍照雙軌、退回補件、請款文件產出、吊車排程整合

---

## 1. 產品目標

### 1.1 第二階段核心目標

- **回報與拍照拆線：** 現場回報不再被拍照作業阻擋，縮短回報時效
- **退回與補件機制：** 監工可退回不合規回報，現場可重新提交
- **請款文件可追溯：** 文件產出依條件篩選，產出時記錄快照與 metadata
- **吊車決策資料化：** 噸數與停駐點資訊整合至彙總看板

### 1.2 成功指標

- 回報完成時間下降 ≥ 30%
- 審核退回次數下降
- 因照片不齊導致回報卡關趨近 0
- 請款補件次數下降

---

## 2. 範圍定義

### 2.1 In Scope

| 項目 | 說明 |
|---|---|
| 回報/拍照雙軌 | need_report / need_photo 控制，各自獨立流程 |
| 退回補件 | 審核者可退回，現場人員可重新回報 |
| 照片歷史版本 | 補拍/換圖保留 Current / History 版本 |
| 文件產出篩選 | 依工序欄位＋狀態篩選，產出含快照 metadata |
| 設備完工回報 | 通知可請款，可重開修改 |
| 吊車欄位整合 | crane_tonnage / parking_zone_group 唯讀顯示 |
| 關帳流程 | 請款送出後方可關帳，關帳後不可變更 |

### 2.2 Out of Scope

- 吊車排程自動化派車（Phase 3 評估）
- 行動裝置離線回報（Phase 3 評估）
- ERP 直接對接（以匯出檔對接）

## 4. 功能需求

### 4.1 回報模組（雙軌）

- 工序主檔新增 need_report / need_photo 旗標
- need_photo = 否 → 不得檢核照片
- need_photo = 是 → 回報可先完成，photo_status = Pending
- PhotoPending 狀態可進入核定/請款流程（與拍照流程分線）

### 4.2 拍照模組

- 照片上傳後 photo_status 轉為 Completed
- 支援補拍與換圖，保留歷史版本（Current / History）
- 關帳後照片不可變更

### 4.3 審核與退回

- 審核者可核定或退回
- 退回時需填寫退回原因
- 退回後回報可重新開啟修改

### 4.4 請款文件產出

- 依工序欄位（need_report / need_photo / parking_zone_group / crane_tonnage）篩選
- 產出時記錄篩選條件與時間戳（快照）
- 文件可重新產出但保留前版紀錄

### 4.5 關帳

- 前置條件：專案經理已送出請款文件
- 關帳後不得變動任何請款資料（含照片/附件）
- 合約關帳，不允許更正

### 4.6 吊車欄位

- crane_tonnage / parking_zone_group 來源為工序主檔
- 回報端顯示為唯讀
- 彙總至看板供主任派車決策

---

## 5. 資料欄位變更

| 欄位 | 類型 | 說明 |
|---|---|---|
| need_report | Boolean | 該工序是否需要回報 |
| need_photo | Boolean | 該工序是否需要拍照 |
| photo_status | Enum | Pending / Completed / NotRequired |
| crane_tonnage | Decimal | 吊車噸數（唯讀） |
| parking_zone_group | String | 停駐點群組（唯讀） |
| completion_report | Object | 設備完工回報資料 |
| close_period | DateTime | 關帳時間 |
| lock | Boolean | 是否已鎖定 |

詳見：[表單與名詞對照清單_欄位引用標準](表單與名詞對照清單_欄位引用標準.md)

---

## 6. 驗收條件

- need_photo = 否 → 回報不得被照片阻擋
- need_photo = 是 → 回報可完成但 photo_status = Pending
- PhotoPending 可核定/請款（與拍照流程分線）
- 退回後可重新回報（回報可再開）
- 文件輸出需記錄篩選條件與產出時間（快照）
- 關帳後不可變更任何資料
