# 工程師開工指引：[主程式代號] [程式名稱]

本指引供取得本交付包後**開工前**與**開發進行**時使用。依 internal_docs/standards/工程師開工指引_治理規範 產出。

---

## 1. 交付包接收者自檢（開工前必做）

取得交付包後、開始實作前，請勾選下列項目：

- [ ] **依賴**：已確認依賴清單（README §3）；依賴之程式／API／TABLE 已取得或可存取，或已知可 mock／暫不實作之範圍
- [ ] **schema／API**：已確認 `03_附錄/schema_參考.md`、`API_規範.md` 存在且涵蓋本程式範圍；若無則已知需自行依欄位對照表與子程式清單產出
- [ ] **驗收條件**：已閱讀 `01_規格/情境與驗收說明.md`，驗收條件為結構化（ID、類型、檢核方式），且可執行檢核
- [ ] **技術棧**：已知本專案技術棧與認證方式（README §4 或專案決策備忘）
- [ ] **環境與權限**：已知建議環境（如下 §3）及取得帳號／連線之方式（由專案負責人提供，不寫入本包機密）

全部勾選後再進行 §2 建議步驟。

---

## 2. 建議進行步驟

1. **讀規格**：依序閱讀 01_規格/PRD_本階段範圍摘要、程式代號與權限矩陣、欄位對照表、情境與驗收說明、子程式清單。
2. **環境準備**：依 §3 建立或確認開發／測試環境；取得依賴程式或 mock 介面。
3. **產 schema／API**：若有 03_附錄/schema_參考、API_規範 則對齊；若無則依欄位對照表與子程式清單產出並可回饋需求方。
4. **依子程式實作**：依子程式清單與實作建議順序展開；每子程式對應元件／API／服務。
5. **驗收檢核**：依情境與驗收說明之驗收條件逐條檢核；標註通過／不通過／待確認。

## 5. AI Prompt 建議（使用 Cursor／AI 時）

可複製以下 Prompt 依情境使用：

- **產 schema**：「請依 01_規格/欄位名稱對照表.md 與子程式清單，產出本程式範圍之 TABLE schema（DDL 或 Markdown 表格式），對齊 internal_docs/模組與程式代碼編碼原則。」
- **產 API 規範**：「請依 01_規格/子程式清單.md 與欄位對照表，產出本程式之 API 路徑與請求／回應格式，對齊 internal_docs/standards/TABLE與API設計_AI產出指引。」
- **依子程式實作**：「請依 01_規格/子程式清單.md 第 [N] 項子程式 [代號]，對應雛型 [區塊]，實作 [元件／API]；命名與欄位依欄位對照表與程式代號。」
- **驗收檢核**：「請依 01_規格/情境與驗收說明.md 驗收條件表，逐條檢核目前實作是否通過，產出通過／不通過／待確認清單。」
