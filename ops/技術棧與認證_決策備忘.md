## 1. 技術棧決策（待工程團隊確認）

| 項目 | 建議選項 | 說明 | 確認 |
| :--- | :--- | :--- | :--- |
| **後端** | .NET 8 / ASP.NET Core | 與 Azure App Service 相容佳；可選 Node.js |  |
| **前端** | React 或 Vue 3 | SPA；可選 Blazor 若與 .NET 統一 |  |
| **ORM** | Entity Framework Core | .NET 生態；可選 Dapper |  |
| **資料庫** | Azure SQL | 與既有 SQL 彈性資料庫一致 |  |
| **API 風格** | REST + JSON |  |

**備註**：依工程團隊既有技術債與偏好調整；確認後請更新本表。

## 2. 認證與授權流程（建議）

| 項目 | 說明 |
| :--- | :--- |
| **登入** | 帳號密碼；密碼不可明文儲存（bcrypt 或 PBKDF2） |
| **Session** | JWT 或 Cookie-based；Session 逾時建議 30 分鐘～2 小時 |
| **登出** | 清除 REDACTED／session；可選 REDACTED 黑名單 |
| **權限檢查** | 依 01_規格/程式代號與權限矩陣.md；前後端皆須驗證 |
| **角色** | 承辦、主管、會計、運務主管等；依權限矩陣表 |

**實作**：可選 ASP.NET Core Identity、JWT Bearer \<REDACTED>；或既有 SSO 整合。
